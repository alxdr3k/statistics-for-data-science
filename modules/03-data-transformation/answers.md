# 03. 데이터 변환 · 확인 문제 해설

## 1. 데이터 변환이 필요한 이유를 “모델이 보는 숫자”와 “사람이 해석하는 의미” 관점에서 설명하라.

모델은 현실 의미보다 숫자 형태를 먼저 본다. 단위가 큰 변수는 거리나 계수 계산에서 더 크게 작용할 수 있고, 문자 범주는 그대로 계산할 수 없다. 사람 입장에서도 원래 값보다 구간, 비율, 로그값이 더 해석하기 쉬운 경우가 있다. 따라서 변환은 모델이 잘 계산하고 사람이 잘 해석하도록 표현을 바꾸는 일이다.

## 2. Min-Max 정규화와 Z-score 표준화의 차이를 설명하라.

Min-Max 정규화는 최솟값을 0, 최댓값을 1로 맞춘다. 값이 전체 범위에서 어디쯤 있는지 보여 준다. Z-score 표준화는 평균을 0, 표준편차를 1로 맞춘다. 값이 평균에서 표준편차 몇 개만큼 떨어져 있는지 보여 준다.

## 3. Min-Max 정규화가 이상치에 민감한 이유를 설명하라.

Min-Max 정규화는 최솟값과 최댓값을 기준으로 전체 범위를 정한다. 매우 큰 이상치 하나가 최댓값이 되면 나머지 정상값들이 0 근처로 압축된다. 그래서 데이터 대부분의 차이가 작아 보일 수 있다.

## 4. 명목형 범주에 레이블 인코딩을 적용하면 어떤 오해가 생길 수 있는가?

명목형 범주에는 크기 순서가 없다. 그런데 `서울=0`, `부산=1`, `대구=2`처럼 숫자를 붙이면 모델이 대구가 부산보다 크고 부산이 서울보다 크다고 오해할 수 있다. 거리 기반 모델에서는 숫자 차이가 실제 거리처럼 쓰일 위험도 있다.

## 5. 타깃 인코딩에서 데이터 누수가 생기는 이유와 이를 줄이는 방법을 설명하라.

타깃 인코딩은 범주를 목표 변수의 평균이나 비율로 바꾼다. 이때 평가해야 할 데이터의 정답 정보가 인코딩 계산에 들어가면 데이터 누수가 된다. 또한 표본이 적은 범주는 우연한 평균을 진짜 규칙처럼 만들 수 있다. 이를 줄이려면 훈련 데이터 안에서만 인코딩을 계산하고, 교차검증 방식이나 평활화를 사용한다.

## 6. 로그 변환이 오른쪽 꼬리를 완화하는 이유를 설명하라.

로그 함수는 값이 커질수록 증가 속도가 느려진다. 그래서 큰 값들의 차이가 원래보다 작게 표현된다. 오른쪽 꼬리에 있는 큰 값들이 중앙 쪽으로 당겨져 보이고, 큰 값 몇 개가 평균이나 모델을 흔드는 힘이 줄어든다.

## 7. Box-Cox와 Yeo-Johnson 변환을 처음 배우는 사람에게 한 문장으로 설명하라.

둘 다 데이터를 더 안정적인 분포에 가깝게 만들기 위해 적절한 거듭제곱 값을 찾는 방법이다. Box-Cox는 양수 데이터에 주로 쓰고, Yeo-Johnson은 0이나 음수가 있어도 사용할 수 있다.

## 8. 구간화의 장점과 정보 손실 위험을 예로 설명하라.

구간화는 연속값을 해석하기 쉬운 범주로 바꾼다. 예를 들어 나이를 `청소년`, `성인`, `노인`으로 나누면 의사결정이 쉬워질 수 있다. 하지만 64세와 65세처럼 경계 근처의 값은 실제로 비슷해도 다른 집단처럼 보일 수 있다. 세밀한 숫자 차이도 사라진다.

## 9. 요약 변수와 파생 변수의 차이를 고객 구매 데이터 예로 설명하라.

요약 변수는 여러 행을 묶어 만든다. 거래 데이터에서 고객별 총 구매액, 구매 횟수, 최근 구매일을 만드는 것이 예다. 파생 변수는 한 행 안의 기존 변수를 조합해 만든다. 예를 들어 상품 가격과 수량으로 결제 금액을 만들거나, 방문 수와 구매 수로 전환율을 만드는 것이 예다.
