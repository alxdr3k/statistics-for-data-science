# 03. 데이터 변환 · 확인 문제 해설

## 해설 읽는 법

기본 해설은 먼저 핵심 문장을 잡고, 스스로 답할 때는 **한 문장 답 → 이유 → 예시 → 주의점** 순서로 확장한다.

## 1. 데이터 변환이 필요한 이유를 “모델이 보는 숫자”와 “사람이 해석하는 의미” 관점에서 설명하라.

모델은 현실 의미보다 숫자 형태를 먼저 본다. 단위가 큰 변수는 거리나 계수 계산에서 더 크게 작용할 수 있고, 문자 범주는 그대로 계산할 수 없다. 사람 입장에서도 원래 값보다 구간, 비율, 로그값이 더 해석하기 쉬운 경우가 있다. 따라서 변환은 모델이 잘 계산하고 사람이 잘 해석하도록 표현을 바꾸는 일이다.

## 2. Min-Max 정규화와 Z-score 표준화의 차이를 설명하라.

Min-Max 정규화는 최솟값을 0, 최댓값을 1로 맞춘다. 값이 전체 범위에서 어디쯤 있는지 보여 준다. Z-score 표준화는 평균을 0, 표준편차를 1로 맞춘다. 값이 평균에서 표준편차 몇 개만큼 떨어져 있는지 보여 준다.

## 3. Min-Max 정규화가 이상치에 민감한 이유를 설명하라.

Min-Max 정규화는 최솟값과 최댓값을 기준으로 전체 범위를 정한다. 매우 큰 이상치 하나가 최댓값이 되면 나머지 정상값들이 0 근처로 압축된다. 그래서 데이터 대부분의 차이가 작아 보일 수 있다.

## 4. 명목형 범주에 레이블 인코딩을 적용하면 어떤 오해가 생길 수 있는가?

명목형 범주에는 크기 순서가 없다. 그런데 `서울=0`, `부산=1`, `대구=2`처럼 숫자를 붙이면 모델이 대구가 부산보다 크고 부산이 서울보다 크다고 오해할 수 있다. 거리 기반 모델에서는 숫자 차이가 실제 거리처럼 쓰일 위험도 있다.

## 5. 타깃 인코딩에서 데이터 누수가 생기는 이유와 이를 줄이는 방법을 설명하라.

타깃 인코딩은 범주를 목표 변수의 평균이나 비율로 바꾼다. 이때 평가해야 할 데이터의 정답 정보가 인코딩 계산에 들어가면 데이터 누수가 된다. 또한 표본이 적은 범주는 우연한 평균을 진짜 규칙처럼 만들 수 있다. 이를 줄이려면 훈련 데이터 안에서만 인코딩을 계산하고, 교차검증 방식이나 평활화를 사용한다.

## 6. 로그 변환이 오른쪽 꼬리를 완화하는 이유를 설명하라.

로그 함수는 값이 커질수록 증가 속도가 느려진다. 그래서 큰 값들의 차이가 원래보다 작게 표현된다. 오른쪽 꼬리에 있는 큰 값들이 중앙 쪽으로 당겨져 보이고, 큰 값 몇 개가 평균이나 모델을 흔드는 힘이 줄어든다.

## 7. Box-Cox와 Yeo-Johnson 변환을 처음 배우는 사람에게 한 문장으로 설명하라.

둘 다 데이터를 더 안정적인 분포에 가깝게 만들기 위해 적절한 거듭제곱 값을 찾는 방법이다. Box-Cox는 양수 데이터에 주로 쓰고, Yeo-Johnson은 0이나 음수가 있어도 사용할 수 있다.

## 8. 구간화의 장점과 정보 손실 위험을 예로 설명하라.

구간화는 연속값을 해석하기 쉬운 범주로 바꾼다. 예를 들어 나이를 `청소년`, `성인`, `노인`으로 나누면 의사결정이 쉬워질 수 있다. 하지만 64세와 65세처럼 경계 근처의 값은 실제로 비슷해도 다른 집단처럼 보일 수 있다. 세밀한 숫자 차이도 사라진다.

## 9. 요약 변수와 파생 변수의 차이를 고객 구매 데이터 예로 설명하라.

요약 변수는 여러 행을 묶어 만든다. 거래 데이터에서 고객별 총 구매액, 구매 횟수, 최근 구매일을 만드는 것이 예다. 파생 변수는 한 행 안의 기존 변수를 조합해 만든다. 예를 들어 상품 가격과 수량으로 결제 금액을 만들거나, 방문 수와 구매 수로 전환율을 만드는 것이 예다.

## 10. 왜 거리 기반 모델에서는 표준화가 중요할 수 있는가?

- **한 문장 답**: 단위가 큰 변수가 거리 계산을 지배할 수 있기 때문이다.
- **이유**: 거리는 각 변수 차이를 합쳐 계산하므로 단위가 큰 변수의 숫자 차이가 실제 중요도보다 크게 반영될 수 있다.
- **예시**: 소득은 만 단위이고 나이는 십 단위라면, 원자료 거리에서는 소득 차이가 거의 모든 것을 결정한다.
- **주의점**: 모든 모델에 무조건 필요한 것은 아니지만 거리나 기울기 기반 모델에서는 먼저 점검한다.

## 11. 왜 로그 변환은 오른쪽 꼬리가 긴 분포를 완화하는가?

- **한 문장 답**: 큰 값일수록 로그를 취했을 때 상대적으로 더 많이 압축되기 때문이다.
- **이유**: 로그는 곱셈적 차이를 덧셈적 차이로 바꾸고 큰 값 사이 간격을 좁힌다. 그래서 긴 꼬리의 극단값 영향이 줄어든다.
- **예시**: 1, 10, 100의 간격은 원래 9와 90이지만 로그에서는 훨씬 가까워진다.
- **주의점**: 0이나 음수에는 그대로 로그를 취할 수 없다.

## 12. 빈도 인코딩과 타깃 인코딩의 차이를 설명하라.

빈도 인코딩은 각 범주가 얼마나 자주 등장하는지를 숫자로 바꾼다. 타깃 인코딩은 각 범주의 목표값 평균이나 성공률을 숫자로 쓴다. 타깃 인코딩은 예측 신호가 강할 수 있지만 정답 정보를 사용하므로 데이터 누수를 특히 조심해야 한다.

## 13. 평활화가 타깃 인코딩에서 필요한 이유를 설명하라.

표본 수가 적은 범주는 우연히 평균이 매우 높거나 낮게 나올 수 있다. 평활화는 이런 범주의 값을 전체 평균 쪽으로 당겨 과신을 줄인다. 표본이 많은 범주는 자기 범주 평균을 더 믿고, 표본이 적은 범주는 전체 평균을 더 섞는 방식이다.

## 14. 데이터 변환의 다섯 종류인 정규화, 인코딩, 분포 변환, 구간화, 요약/파생 변수를 각각 한 문장으로 구분하라.

정규화는 서로 다른 단위와 범위를 공통 척도로 맞추는 변환이다. 인코딩은 비정형 또는 비수치형 데이터를 모델이 계산할 수 있는 수치형 표현으로 바꾸는 과정이다. 분포 변환은 정규성, 학습 안정성, 이상치 영향 완화 등을 위해 분포 모양을 바꾸는 작업이다. 구간화는 연속형 변수를 범주형 변수로 나누는 작업이다. 요약 변수는 여러 행을 묶어 만든 변수이고, 파생 변수는 기존 변수를 조합해 새로 만든 변수다.

## 15. 단순 거듭제곱 변환에서 `lambda > 1`과 `0 < lambda < 1`이 분포에 주는 효과를 비교하라.

`lambda > 1`이면 큰 값이 더 커져 값 사이 차이가 벌어진다. 왼쪽으로 긴 꼬리를 완화하거나 큰 값의 차이를 더 드러내고 싶을 때 고려할 수 있다. `0 < lambda < 1`이면 큰 값이 압축되어 오른쪽 꼬리가 짧아진다. 제곱근 변환처럼 우측 왜도를 줄이고 싶을 때 자주 쓰인다.

## 16. Box-Cox 변환의 조각별 수식을 쓰고, `lambda = 0`일 때 왜 로그 변환이 되는지 설명하라.

Box-Cox 변환은 양수 데이터에서 다음처럼 쓴다.

```text
y_i(lambda) = (y_i^lambda - 1) / lambda,  if lambda != 0
            = ln(y_i),                    if lambda = 0
```

`lambda`가 0이 아닐 때는 거듭제곱 형태이고, `lambda = 0`일 때는 로그 변환으로 정의한다. 수학적으로도 `(y^lambda - 1)/lambda`는 `lambda`가 0에 가까워질 때 `ln(y)`에 가까워진다. 그래서 Box-Cox는 로그 변환을 포함하는 거듭제곱 변환 계열로 이해할 수 있다.

## 17. Yeo-Johnson 변환이 Box-Cox와 달리 0이나 음수에도 적용될 수 있는 이유를 설명하라.

Box-Cox는 `ln(y)`와 `y^lambda`를 쓰므로 기본적으로 `y`가 양수여야 한다. Yeo-Johnson은 `y >= 0`일 때는 `y+1`을 사용하고, `y < 0`일 때는 `-y+1`을 사용해 양수 형태로 바꾼 뒤 부호를 조정한다. 그래서 0이나 음수가 포함된 변수에도 적용할 수 있다.

## 18. 파생 변수에 대해 “특정 상황에서만 유의미하면 충분하다”는 설명이 왜 부적절한지 설명하라.

파생 변수는 기존 변수를 조합해 만든 새 변수이므로 분석 목적과 업무 논리가 있어야 한다. 특정 데이터 상황에서만 우연히 좋아 보이는 변수는 훈련 데이터에 과하게 맞은 신호일 수 있다. 좋은 파생 변수는 특정 표본의 우연보다, 반복되는 상황에서도 해석 가능한 대표성과 논리성을 가져야 한다.

## 19. 0 이상의 큰 이상치에 대응할 때 Min-Max 정규화가 로그 변환이나 구간화보다 덜 적합할 수 있는 이유를 설명하라.

Min-Max 정규화는 최솟값과 최댓값을 기준으로 값을 0과 1 사이로 옮긴다. 큰 이상치가 최댓값이 되면 나머지 대부분의 값이 0 근처로 눌려 분포가 왜곡될 수 있다. 반면 로그 변환은 큰 값을 더 강하게 압축하고, 구간화는 극단값을 같은 상위 구간으로 묶어 영향력을 줄일 수 있다. 그래서 “0 이상의 극단적으로 큰 값”에 대응하는 목적이라면 Min-Max 정규화는 가장 거리가 먼 선택일 수 있다.
