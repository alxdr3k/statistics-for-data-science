# 23. 종합 문제 1

## 학습 목표

이 차시는 1~22차시의 개념을 섞어 묻는 첫 번째 종합 복습이다. 단어 하나를 기억하는 것보다, 문제가 어떤 차시의 판단 기준을 요구하는지 먼저 찾는 연습을 한다.

- 데이터 이해, 전처리, 변환 문제에서 먼저 봐야 할 기준을 구분한다.
- 검정, 모델 평가, 일반화 문제에서 올바른 판단 기준을 고르는 이유를 설명한다.
- 머신러닝과 신경망 문제를 “모델 이름”이 아니라 작동 방식과 한계로 판단한다.

## 오늘의 한 줄

종합 문제는 많이 아는지를 묻기보다, 섞여 있는 단서에서 어떤 기준을 먼저 꺼내야 하는지를 묻는다.

## 오늘 반드시 이해할 3가지

1. 데이터 문제는 `데이터 형태 -> 처리 목적 -> 왜곡 위험` 순서로 본다.
2. 모델 문제는 `학습 방식 -> 일반화 위험 -> 성능 지표` 순서로 본다.
3. 통계 문제는 `모수/표본 -> 검정 대상 -> 기준 분포` 순서로 본다.

## 종합 복습 주제

이 차시는 다음 주제를 섞어 확인한다.

- 빅데이터의 특징과 실시간 분석 가능성
- 범주형 빈도표와 성공률 비교
- 상호작용 차트 해석
- 변수 처리와 로그 변환의 적합 상황
- 확률과 분포의 기본 판단
- 교차검증 분리 방식
- 규제 손실함수 해석
- 이상치 탐색법
- KDD와 CRISP-DM 비교
- 군집화, 경사하강법, 가설검정 방법 선택
- 랜덤 포레스트, 앙상블, 과적합 대응
- 기울기 소실과 신경망 구조
- 희귀질환 예측 지표
- PCA와 표본 추출법

## 문제를 푸는 순서

### 1. 단어보다 문제 유형을 먼저 찾기

예를 들어 `HomePlanet`, `Destination`, `Transported rate`가 같이 나오면 단순히 변수 이름을 외우는 문제가 아니다. 두 범주형 변수의 조합에 따라 대상 비율이 어떻게 달라지는지 보는 문제다. 이때는 교차표, 비율, 상호작용 차트가 먼저 떠올라야 한다.

반대로 `희귀암 예측`처럼 양성 클래스가 매우 드문 상황은 정확도만 보면 위험하다. 대부분을 음성으로 찍어도 정확도가 높아질 수 있기 때문이다. 이때는 재현율, 정밀도, F1, ROC-AUC, PR-AUC처럼 불균형 문제에 맞는 지표를 떠올린다.

### 2. 전처리 문제는 목적과 부작용을 같이 보기

로그 변환은 큰 값의 영향이 지나치게 강하거나 오른쪽 꼬리가 긴 양수 데이터에서 유용하다. 하지만 0 이하 값이 많거나, 해석상 비율 변화가 의미 없거나, 이미 대칭에 가까운 데이터라면 무리하게 적용할 이유가 약하다.

이상치 탐색도 마찬가지다. IQR은 단변량 기준이 강하고, 산점도는 두 변수의 관계를 보기 좋으며, LOF와 Isolation Forest는 다변량 구조에서 튀는 점을 찾는 데 쓴다. 어떤 방법이 좋다는 말보다 “무엇을 이상하다고 볼 것인가”가 먼저다.

### 3. 모델 문제는 일반화 관점으로 읽기

랜덤 포레스트는 부트스트랩 표본과 무작위 특성 선택으로 여러 의사결정나무를 만든다. 이 조합은 개별 나무의 분산을 낮추는 데 도움이 된다. 반면 부스팅은 이전 모델의 오차를 다음 모델이 보완하는 순차적 접근이다.

경사하강법 문제에서는 손실이 항상 매끄럽게 줄어든다고 가정하면 틀리기 쉽다. 확률적 또는 미니배치 방식은 일부 표본만 보고 기울기를 추정하므로 손실이 흔들리면서 내려갈 수 있다.

## 세부 판단 포인트

빅데이터는 정형, 반정형, 비정형 데이터를 모두 포함할 수 있고, 대용량·고속 처리·다양성 같은 특징을 가진다. “수집 당시 데이터만 분석할 수 있으므로 실시간 분석은 불가능하다”는 단정은 틀리다. 적절한 스트리밍 처리 시스템과 알고리즘이 있으면 실시간 분석도 가능하다.

범주형 빈도표의 카이제곱 독립성 검정에서는 자유도와 기대빈도를 먼저 계산한다. 행이 `A, B, C` 3개이고 열이 `남자, 여자` 2개이면 자유도는 `(3 - 1) x (2 - 1) = 2`다. 표의 전체 합계가 `100`, 여자 합계가 `40`, `X=B` 행 합계가 `30`이면 `G=여자, X=B` 기대빈도는 `30 x 40 / 100 = 12`다. 카이제곱 검정은 관측빈도와 기대빈도의 차이가 클수록 통계량이 커지므로 대립가설을 지지하는 영역은 우측 꼬리에 놓인다.

두 범주형 변수의 상호작용은 각 변수의 개별 평균만으로 조합의 결과를 설명할 수 없는 경우를 말한다. 예를 들어 `HomePlanet=Mars`도, `Destination=PSO J318-5-22`도 각각은 중간 정도 성공률인데 두 조건이 동시에 나타날 때 가장 낮은 성공률을 보인다면 두 변수를 독립적으로만 설명하기 어렵다.

변수 처리에서는 변환 목적과 부작용을 같이 본다. 절편이 있는 선형회귀에서 범주형 변수를 가변수화할 때는 기준 범주 하나를 제외해야 다중공선성을 피할 수 있다. 표준정규화를 하면 원래 변수 단위는 사라진다. 낮은 빈도의 범주 통계치를 안정화할 때는 smoothing을 쓸 수 있다. 반면 Min-Max 정규화는 최솟값과 최댓값에 직접 의존하므로 이상치에 강하지 않다.

로그 변환은 큰 값의 영향 완화, 등분산성 개선, 평균 비교나 이상치 영향 완화에는 도움이 될 수 있다. 그러나 Spearman 상관은 값의 순위에 대한 Pearson 상관이고, 로그 함수는 단조 증가 함수라 순위를 바꾸지 않는다. 따라서 순위 상관 자체를 바꾸려는 목적이라면 로그 변환의 의미가 약하다.

추정에서는 불편추정량과 적률생성함수를 구분한다. 불편추정량은 `E(theta_hat) = theta`처럼 추정량의 기대값이 참 모수와 일치하는 추정량이다. 적률생성함수는 미분을 통해 평균, 분산 같은 적률을 얻는 데 쓰이고, 적률법 점추정의 배경이 된다.

교차검증에서 계층적 분리(stratified split)는 클래스 비율이 치우친 경우 각 fold의 클래스 비율을 비슷하게 유지하기 위한 방법이다. 평가셋이 학습셋에 등장하지 않은 케이스로 구성되는 문제는 그룹 분리나 시간·대상 단위 분리를 먼저 의심해야 한다.

Ridge와 Lasso는 손실에 더하는 규제항이 다르다. Ridge는 L2 규제로 큰 회귀계수를 전반적으로 억제하고, Lasso는 L1 규제로 일부 계수를 0으로 만들어 속성 선택 효과를 낼 수 있다. 두 규제를 함께 쓰면 Elastic Net이다. Ridge는 해석적 계수 유도가 가능하므로 “해석적으로 계수를 구할 수 없다”는 설명은 맞지 않는다.

이상치 탐색에서 DBSCAN은 이상치를 “가장 작은 군집”으로 보는 것이 아니라 어떤 군집에도 속하지 않는 noise로 본다. Isolation Forest는 이상치가 적은 분기만으로 고립된다고 가정하고, LOF는 주변보다 상대 밀도가 낮은 점을 이상치로 본다. PCA 기반 방법은 주성분들로 설명되지 않는 정보량이 큰 점을 의심한다.

절차 방법론에서는 KDD와 CRISP-DM의 시작점이 다르다. KDD는 데이터셋 선택, 전처리, 변환, 데이터 마이닝, 해석 및 평가 흐름이다. CRISP-DM은 비즈니스 이해가 먼저이고, 데이터 이해, 데이터 준비, 모델링, 평가, 전개로 이어진다.

군집화에서는 대표점의 의미를 조심한다. k-means의 대표는 평균 중심점이고, k-medoids의 대표는 실제 데이터 포인트 중 군집 내 다른 점들과의 거리 합이 가장 작은 점이다.

```text
m_k = argmin_{x_j in C_k} sum_{x_i in C_k} distance(x_i, x_j)
```

경사하강법에서는 배치 크기가 커지면 한 번의 기울기 추정이 전체 경사도를 더 잘 대표하므로 안정성은 보통 높아진다. 따라서 “배치를 키우면 안정성이 낮아진다”는 설명은 맞지 않다. NAG는 예측한 위치 `theta - gamma * v_(t-1)`에서 기울기를 계산하고, Adagrad는 이동이 적거나 희소한 방향의 학습을 상대적으로 돕는다.

같은 사람의 복용 전후 몸무게처럼 대응 관계가 있는 데이터는 paired t 검정이 자연스럽다. 독립표본 t 검정도 계산 자체는 가능할 수 있지만, 사람별 전후 차이를 직접 쓰는 것이 자료 구조에 더 맞다.

Dropout은 학습 중 임의의 노드를 제외해 매번 조금 다른 부분 신경망을 학습하는 효과를 낸다. 이 점에서 부트스트랩으로 다양한 의사결정나무를 만드는 랜덤 포레스트의 다양성 확보와 비슷한 직관을 가진다.

앙상블에서 Bagging은 Bootstrap과 Aggregation의 결합이다. AdaBoost는 손실 감소 정도에 따라 학습률을 자동 조정한다기보다, 오답 샘플의 학습 가중치를 높여 다음 약한 학습기가 더 집중하게 만든다. AUC처럼 순위 정보가 중요한 지표에서는 hard voting보다 확률 점수를 평균하는 soft voting이 더 적합한 경우가 많다.

기울기 소실은 출력층 가까이가 아니라 입력층 쪽으로 갈수록 더 심해진다. 역전파가 출력층에서 입력층 방향으로 진행되고, 작은 편미분들이 연쇄적으로 곱해지기 때문이다. RNN의 게이트, ResNet의 잔차 연결, Batch Normalization은 이 문제를 완화하는 방향의 구조이고, Dropout은 주로 과적합 대응 장치다.

Word2Vec의 CBOW 구조는 주변 단어를 입력으로 받아 중심 단어를 예측한다. 핵심 결과물은 단어 의미를 담은 임베딩 벡터이며, 사전학습 표현으로 활용할 수 있다. 임베딩 시각화에는 차원축소를 쓸 수 있지만, PLS는 정답 변수가 있는 경우에 쓰는 지도 방식이므로 이 설명에는 맞지 않는다.

희귀암처럼 양성 클래스가 드문 문제에서 F1을 개선하려면 양성 클래스의 표본 수를 늘리거나, 양성 클래스 가중치를 높이거나, 임계값을 조정하는 접근이 가능하다. 반대로 양성 데이터 포인트를 제거해 불균형을 더 키우는 처리는 부적절하다.

PCA에서는 공분산행렬의 고유값이 해당 주성분 방향의 분산을 나타내고, SVD로도 PCA 변환을 유도할 수 있다. 유클리디언 거리 기반의 고전적 MDS는 PCA와 연결될 수 있지만, 마할라노비스 거리는 공분산행렬의 역행렬로 정규화한 거리이므로 PCA 변환 후 유클리디언 거리와 항상 같다고 말할 수 없다.

표본 추출에서는 층화추출, 집락추출, 계통추출을 구분한다. 층화추출은 동질적인 층으로 나눈 뒤 각 층에서 무작위로 뽑고, 집락추출은 군집 일부를 뽑는다. 일정한 간격으로 표본을 뽑는 방식은 집락추출이 아니라 계통추출이다.

## 대표 판단 기준

| 문제 단서 | 먼저 떠올릴 기준 | 답안에 포함할 말 |
|---|---|---|
| 빅데이터 3V/5V | 규모, 속도, 다양성, 정확성, 가치 | 실시간 분석 가능 여부는 시스템과 처리 방식에 달려 있다. |
| 빈도표, 성공률 | 교차표, 조건부 비율, 상호작용 | 개별 변수 효과와 조합 효과를 구분한다. |
| 로그 변환 | 오른쪽 꼬리, 양수, 큰 값 완화 | 0 이하 값과 순위 기반 지표에는 주의한다. |
| 교차검증 | 데이터 누수와 분리 단위 | 행 단위가 아니라 사람, 시간, 그룹 단위 분리가 필요할 수 있다. |
| 규제 손실함수 | L1/L2, 복잡도 벌점 | Ridge는 계수를 줄이고 Lasso는 일부 계수를 0으로 만들 수 있다. |
| 희귀질환 예측 | 클래스 불균형 지표 | 정확도만으로는 양성 탐지 성능을 판단하기 어렵다. |
| PCA | 분산 보존, 주성분 해석 | 새 축을 만드는 차원 축소라 원래 변수 선택과 다르다. |
| 표본 추출 | 확률/비확률 표본 추출 구분 | 층화, 집락, 계통 추출의 뽑는 단위가 다르다. |

답안을 쓸 때는 방법 이름만 쓰지 말고 조건도 같이 적는다. 예를 들어 “paired t 검정”이라고만 쓰면 부족하고, 같은 사람의 전후 측정이라 관측값이 짝을 이룬다는 점을 함께 말해야 한다. “F1을 본다”라고만 쓰는 것도 부족하다. 양성 클래스가 드물어 정확도가 양성 탐지 실패를 가릴 수 있다는 이유를 붙여야 한다.

## 예시 풀이

### 예시 1. 로그 변환이 어울리지 않는 경우

값이 모두 양수이고 오른쪽 꼬리가 길면 로그 변환은 큰 값의 영향력을 줄인다. 그러나 값이 0 또는 음수를 포함하면 그대로 로그를 취할 수 없다. 또한 이미 거의 대칭인 분포라면 로그 변환이 오히려 분포를 왜곡할 수 있다.

### 예시 2. 희귀질환 예측 모델 평가

희귀질환처럼 양성 비율이 낮은 문제에서는 정확도만으로 모델을 평가하면 위험하다. 양성을 놓치는 비용이 크다면 재현율을 먼저 보고, 양성 판정의 신뢰도까지 필요하면 정밀도와 F1을 함께 본다.

## 확인 문제

1. 빅데이터에서 실시간 분석이 불가능하다는 말이 왜 지나친 단정인지 설명하라.
2. 카이제곱 독립성 검정에서 자유도와 기대빈도를 계산하는 방법을 설명하라.
3. 두 범주형 변수의 조합이 성공률을 따로 설명할 수 없게 만드는 상황을 설명하라.
4. Min-Max 정규화가 이상치에 민감한 이유를 설명하라.
5. 로그 변환이 Spearman 상관에는 큰 변화를 만들기 어려운 이유를 설명하라.
6. 불편추정량과 적률생성함수의 역할을 구분하라.
7. 클래스 불균형이 있을 때 층화 분리가 필요한 이유를 설명하라.
8. Ridge, Lasso, Elastic Net의 차이를 설명하라.
9. DBSCAN에서 이상치가 작은 군집이 아니라 noise로 남는다는 뜻을 설명하라.
10. KDD와 CRISP-DM의 올바른 절차 차이를 설명하라.
11. k-means와 k-medoids의 대표점 차이를 설명하라.
12. 배치 크기가 커질 때 경사 추정의 안정성이 어떻게 달라지는지 설명하라.
13. 같은 사람의 전후 측정에서 paired t 검정을 쓰는 이유를 설명하라.
14. Dropout이 랜덤 포레스트의 다양성과 비슷한 효과를 내는 이유를 설명하라.
15. Bagging, AdaBoost, Soft voting을 구분하라.
16. 기울기 소실이 입력층 쪽에서 더 심해지는 이유를 설명하라.
17. Word2Vec이 사전학습 표현으로 유용한 이유를 설명하라.
18. 불균형 데이터에서 F1을 개선하려면 어떤 방향의 처리가 필요한지 설명하라.
19. PCA, SVD, MDS, Mahalanobis 거리의 관계를 조심해서 설명하라.
20. 집락추출과 계통추출을 구분하라.
21. 종합 문제 답안에서 방법 이름만 쓰지 않고 조건과 이유를 함께 적어야 하는 이유를 설명하라.
22. 교차검증 문제에서 행 단위 분리가 항상 충분하지 않을 수 있는 이유를 설명하라.
23. 희귀질환 예측에서 정확도가 높아도 좋은 모델이라고 단정하기 어려운 이유를 설명하라.

## 오늘의 요약 5줄

1. 종합 문제는 문제 단서가 어느 차시의 판단 기준을 요구하는지 찾는 연습이다.
2. 전처리 문제는 변환 목적과 정보 왜곡 위험을 함께 본다.
3. 모델 문제는 학습 방식, 일반화, 성능 지표를 함께 본다.
4. 불균형 데이터에서는 정확도보다 재현율, 정밀도, F1, PR-AUC가 중요할 수 있다.
5. 표본 추출, PCA, 검정, 앙상블은 이름보다 “왜 그 방법을 쓰는가”를 설명해야 한다.
