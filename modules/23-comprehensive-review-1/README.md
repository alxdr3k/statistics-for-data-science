# 23. 종합 문제 1

## 학습 목표

이 차시는 1~22차시의 개념을 섞어 묻는 첫 번째 종합 복습이다. 단어 하나를 기억하는 것보다, 문제가 어떤 차시의 판단 기준을 요구하는지 먼저 찾는 연습을 한다.

- 데이터 이해, 전처리, 변환 문제에서 먼저 봐야 할 기준을 구분한다.
- 검정, 모델 평가, 일반화 문제에서 올바른 선택지를 고르는 이유를 설명한다.
- 머신러닝과 신경망 문제를 “모델 이름”이 아니라 작동 방식과 한계로 판단한다.

## 오늘의 한 줄

종합 문제는 많이 아는지를 묻기보다, 섞여 있는 단서에서 어떤 기준을 먼저 꺼내야 하는지를 묻는다.

## 오늘 반드시 이해할 3가지

1. 데이터 문제는 `데이터 형태 -> 처리 목적 -> 왜곡 위험` 순서로 본다.
2. 모델 문제는 `학습 방식 -> 일반화 위험 -> 성능 지표` 순서로 본다.
3. 통계 문제는 `모수/표본 -> 검정 대상 -> 기준 분포` 순서로 본다.

## 종합 복습 주제

이 차시는 다음 주제를 섞어 확인한다.

- 빅데이터의 특징과 실시간 분석 가능성
- 범주형 빈도표와 성공률 비교
- 상호작용 차트 해석
- 변수 처리와 로그 변환의 적합 상황
- 확률과 분포의 기본 판단
- 교차검증 분리 방식
- 규제 손실함수 해석
- 이상치 탐색법
- KDD와 CRISP-DM 비교
- 군집화, 경사하강법, 가설검정 방법 선택
- 랜덤 포레스트, 앙상블, 과적합 대응
- 기울기 소실과 신경망 구조
- 희귀질환 예측 지표
- PCA와 표본 추출법

## 문제를 푸는 순서

### 1. 단어보다 문제 유형을 먼저 찾기

예를 들어 `HomePlanet`, `Destination`, `Transported rate`가 같이 나오면 단순히 변수 이름을 외우는 문제가 아니다. 두 범주형 변수의 조합에 따라 대상 비율이 어떻게 달라지는지 보는 문제다. 이때는 교차표, 비율, 상호작용 차트가 먼저 떠올라야 한다.

반대로 `희귀암 예측`처럼 양성 클래스가 매우 드문 상황은 정확도만 보면 위험하다. 대부분을 음성으로 찍어도 정확도가 높아질 수 있기 때문이다. 이때는 재현율, 정밀도, F1, ROC-AUC, PR-AUC처럼 불균형 문제에 맞는 지표를 떠올린다.

### 2. 전처리 문제는 목적과 부작용을 같이 보기

로그 변환은 큰 값의 영향이 지나치게 강하거나 오른쪽 꼬리가 긴 양수 데이터에서 유용하다. 하지만 0 이하 값이 많거나, 해석상 비율 변화가 의미 없거나, 이미 대칭에 가까운 데이터라면 무리하게 적용할 이유가 약하다.

이상치 탐색도 마찬가지다. IQR은 단변량 기준이 강하고, 산점도는 두 변수의 관계를 보기 좋으며, LOF와 Isolation Forest는 다변량 구조에서 튀는 점을 찾는 데 쓴다. 어떤 방법이 좋다는 말보다 “무엇을 이상하다고 볼 것인가”가 먼저다.

### 3. 모델 문제는 일반화 관점으로 읽기

랜덤 포레스트는 부트스트랩 표본과 무작위 특성 선택으로 여러 의사결정나무를 만든다. 이 조합은 개별 나무의 분산을 낮추는 데 도움이 된다. 반면 부스팅은 이전 모델의 오차를 다음 모델이 보완하는 순차적 접근이다.

경사하강법 문제에서는 손실이 항상 매끄럽게 줄어든다고 가정하면 틀리기 쉽다. 확률적 또는 미니배치 방식은 일부 표본만 보고 기울기를 추정하므로 손실이 흔들리면서 내려갈 수 있다.

## 대표 판단 기준

| 문제 단서 | 먼저 떠올릴 기준 | 연결 차시 |
|---|---|---|
| 빅데이터 3V/5V | 규모, 속도, 다양성, 정확성, 가치 | 1차시 |
| 빈도표, 성공률 | 교차표, 조건부 비율, 상호작용 | 1, 10차시 |
| 로그 변환 | 오른쪽 꼬리, 양수, 큰 값 완화 | 3차시 |
| 교차검증 | 데이터 누수와 분리 단위 | 11차시 |
| 규제 손실함수 | L1/L2, 복잡도 벌점 | 12차시 |
| 희귀질환 예측 | 클래스 불균형 지표 | 11, 18차시 |
| PCA | 분산 보존, 주성분 해석 | 17차시 |
| 표본 추출 | 확률/비확률 표본 추출 구분 | 4차시 |

## 예시 풀이

### 예시 1. 로그 변환이 어울리지 않는 경우

값이 모두 양수이고 오른쪽 꼬리가 길면 로그 변환은 큰 값의 영향력을 줄인다. 그러나 값이 0 또는 음수를 포함하면 그대로 로그를 취할 수 없다. 또한 이미 거의 대칭인 분포라면 로그 변환이 오히려 분포를 왜곡할 수 있다.

### 예시 2. 희귀질환 예측 모델 평가

희귀질환처럼 양성 비율이 낮은 문제에서는 정확도만으로 모델을 평가하면 위험하다. 양성을 놓치는 비용이 크다면 재현율을 먼저 보고, 양성 판정의 신뢰도까지 필요하면 정밀도와 F1을 함께 본다.

## 확인 문제

1. 빅데이터에서 실시간 분석이 불가능하다는 말이 왜 지나친 단정인지 설명하라.
2. 카이제곱 독립성 검정에서 자유도와 기대빈도를 계산하는 방법을 설명하라.
3. 두 범주형 변수의 조합이 성공률을 따로 설명할 수 없게 만드는 상황을 설명하라.
4. Min-Max 정규화가 이상치에 민감한 이유를 설명하라.
5. 로그 변환이 Spearman 상관에는 큰 변화를 만들기 어려운 이유를 설명하라.
6. 불편추정량과 적률생성함수의 역할을 구분하라.
7. 클래스 불균형이 있을 때 층화 분리가 필요한 이유를 설명하라.
8. Ridge, Lasso, Elastic Net의 차이를 설명하라.
9. DBSCAN에서 이상치가 작은 군집이 아니라 noise로 남는다는 뜻을 설명하라.
10. KDD와 CRISP-DM의 올바른 절차 차이를 설명하라.
11. k-means와 k-medoids의 대표점 차이를 설명하라.
12. 배치 크기가 커질 때 경사 추정의 안정성이 어떻게 달라지는지 설명하라.
13. 같은 사람의 전후 측정에서 paired t 검정을 쓰는 이유를 설명하라.
14. Dropout이 랜덤 포레스트의 다양성과 비슷한 효과를 내는 이유를 설명하라.
15. Bagging, AdaBoost, Soft voting을 구분하라.
16. 기울기 소실이 입력층 쪽에서 더 심해지는 이유를 설명하라.
17. Word2Vec이 사전학습 표현으로 유용한 이유를 설명하라.
18. 불균형 데이터에서 F1을 개선하려면 어떤 방향의 처리가 필요한지 설명하라.
19. PCA, SVD, MDS, Mahalanobis 거리의 관계를 조심해서 설명하라.
20. 집락추출과 계통추출을 구분하라.

## 오늘의 요약 5줄

1. 종합 문제는 문제 단서가 어느 차시의 판단 기준을 요구하는지 찾는 연습이다.
2. 전처리 문제는 변환 목적과 정보 왜곡 위험을 함께 본다.
3. 모델 문제는 학습 방식, 일반화, 성능 지표를 함께 본다.
4. 불균형 데이터에서는 정확도보다 재현율, 정밀도, F1, PR-AUC가 중요할 수 있다.
5. 표본 추출, PCA, 검정, 앙상블은 이름보다 “왜 그 방법을 쓰는가”를 설명해야 한다.
