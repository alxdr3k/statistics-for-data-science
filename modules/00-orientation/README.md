# 00. 학습 방법

## 목표

이 커리큘럼은 실습형 튜토리얼이 아니라 이론 강의안입니다. 따라서 “코드가 돌아간다”보다 “왜 이 모델을 써도 되는가를 설명한다”를 목표로 둡니다.

## 권장 루틴

1. 한 차시를 처음 읽을 때는 수식을 건너뛰지 않습니다.
2. 각 수식에서 입력, 모수, 통계량, 손실, 확률을 구분합니다.
3. `판단 기준`을 자기 말로 바꿔 적습니다.
4. `오해와 반례`를 읽고, 실제 데이터에서 어떻게 나타날지 예를 듭니다.
5. 확인 문제는 답안을 쓰기보다 먼저 말로 설명합니다.

## 이론 노트 작성법

각 차시를 읽은 뒤 다음 다섯 줄을 남기면 충분합니다.

```text
오늘의 핵심 개념:
필요한 전제:
대표 수식과 의미:
깨지기 쉬운 오해:
다음 차시와 연결:
```
