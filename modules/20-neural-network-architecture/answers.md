# 20. 신경망의 구성 · 확인 문제 해설

## 해설 읽는 법

기본 해설은 먼저 핵심 문장을 잡고, 스스로 답할 때는 **한 문장 답 → 이유 → 예시 → 주의점** 순서로 확장한다.

## 1. Dense Layer의 파라미터 수를 설명하라.

Dense Layer는 모든 입력 뉴런이 모든 출력 뉴런과 연결된다. 입력 차원이 m이고 출력 뉴런 수가 n이면 가중치는 m x n개이고, 편향은 n개다. 따라서 전체 파라미터 수는 m x n + n이다.

## 2. Embedding이 단순 라벨 인코딩과 다른 이유를 설명하라.

라벨 인코딩은 범주에 번호를 붙일 뿐이라 번호 사이의 의미 관계를 배우지 않는다. Embedding은 각 범주를 여러 차원의 벡터로 바꾸고 그 벡터를 학습한다. 비슷한 단어나 범주는 가까운 표현을 갖도록 학습될 수 있다.

## 3. CNN의 필터가 하는 일을 설명하라.

필터는 이미지의 작은 영역을 훑으면서 특정 패턴이 있는지 감지한다. 낮은 층에서는 선이나 색 변화 같은 단순 패턴을, 깊은 층에서는 더 복잡한 조합을 잡을 수 있다. 같은 필터를 여러 위치에 적용하기 때문에 위치가 조금 달라도 패턴을 찾기 쉽다.

## 4. RNN이 순서 데이터에 적합한 이유를 설명하라.

RNN은 현재 입력뿐 아니라 이전 시점의 상태를 함께 사용한다. 그래서 문장, 시계열처럼 앞의 정보가 뒤의 판단에 영향을 주는 데이터에 맞다. 다만 긴 의존성에서는 기울기 문제가 생길 수 있다.

## 5. LSTM과 GRU가 단순 RNN의 어떤 문제를 줄이려는지 설명하라.

단순 RNN은 긴 시퀀스에서 오래전 정보를 유지하기 어렵고 기울기 소실이 생기기 쉽다. LSTM과 GRU는 게이트 구조로 어떤 정보를 기억하고 버릴지 조절한다. 이 덕분에 더 긴 의존성을 다루기 쉬워진다.

## 6. Dropout이 과대적합을 줄이는 직관을 설명하라.

Dropout은 훈련 중 일부 뉴런을 임시로 꺼서 특정 뉴런 조합에만 의존하지 못하게 한다. 모델은 여러 부분 조합으로도 예측하도록 학습한다. 이 효과가 앙상블처럼 작동해 과대적합을 줄일 수 있다.

## 7. 왜 이미지에는 Dense보다 CNN이 자연스러운 경우가 많은가?

- **한 문장 답**: 이미지는 가까운 픽셀끼리 만드는 지역 패턴이 중요하기 때문이다.
- **이유**: Dense는 픽셀 위치 구조를 펼쳐 버려 지역성과 위치 공유를 직접 활용하기 어렵다. CNN은 작은 필터를 여러 위치에 재사용해 이미지의 공간 패턴을 효율적으로 잡는다.
- **예시**: 고양이 귀나 눈처럼 작은 필터가 찾는 패턴을 여러 위치에서 공유할 수 있다.
- **주의점**: 모든 문제에 CNN이 최선은 아니며 데이터 구조를 먼저 본다.

## 8. 왜 Dropout은 과적합을 줄이는 데 도움이 되는가?

- **한 문장 답**: 학습 중 일부 뉴런을 무작위로 제외해 특정 뉴런 조합에 지나치게 의존하지 않게 만들기 때문이다.
- **이유**: 매번 일부 뉴런이 빠지면 모델은 특정 경로 하나에만 기대기 어렵다. 여러 부분 표현이 함께 작동하도록 압박해 새 데이터에서 덜 흔들리게 한다.
- **예시**: 매번 조금 다른 부분 네트워크로 학습하는 효과가 생긴다.
- **주의점**: 추론 때는 훈련 때처럼 무작위로 계속 버리는 방식이 아니다.

## 9. CNN 출력 크기에 영향을 주는 요소를 설명하라.

입력 크기, 필터 크기, stride, padding이 출력의 공간 크기를 정한다. 필터가 클수록 한 번에 보는 영역이 넓고, stride가 클수록 출력 크기는 작아진다. padding을 넣으면 가장자리 정보를 보존하고 출력 크기 감소를 줄일 수 있다.

## 10. RNN의 many-to-one, one-to-many, many-to-many 구조를 예와 함께 설명하라.

Many-to-one은 여러 시점 입력을 보고 하나의 출력을 내는 구조로 문장 감성 분류가 예다. One-to-many는 하나의 입력에서 여러 출력을 만드는 구조로 이미지 설명 생성처럼 생각할 수 있다. Many-to-many는 입력과 출력이 모두 시퀀스인 구조로 번역이나 시계열 예측에 쓰인다.

## 11. Embedding Layer에서 `Unknown -> 0` 같은 인덱스와 임베딩 테이블이 어떻게 쓰이는지 설명하라.

Embedding Layer의 입력은 범주를 나타내는 정수 인덱스다. 예를 들어 사과를 1, 배를 2, 모르는 범주를 0으로 둘 수 있다. 임베딩 테이블은 `범주 수 k x 임베딩 크기 n` 행렬이고, 입력 인덱스가 가리키는 행의 벡터를 반환한다. 새로 등장한 범주나 저빈도 범주는 충분히 학습된 벡터가 없어서 성능이 낮을 수 있다.

## 12. 다채널 합성곱의 파라미터 수를 계산하는 방법을 설명하라.

커널 높이와 너비, 입력 채널 수, 출력 채널 수를 모두 곱해 커널 파라미터 수를 구하고, 출력 채널 수만큼 편향을 더한다. 식은 `커널 높이 x 커널 너비 x C_in x C_out + C_out`이다. 예를 들어 `3 x 3` 커널, 입력 채널 3, 출력 채널 4라면 `3 x 3 x 3 x 4 + 4 = 112`개다.

## 13. 입력 23, 커널 5, padding 2, stride 2인 합성곱 출력 크기를 계산하라.

출력 크기 공식은 `O = (I - K + 2P) / S + 1`이다. 값을 넣으면 `O = (23 - 5 + 2 x 2) / 2 + 1 = 22 / 2 + 1 = 12`다. 따라서 2차원 입력이면 출력 공간 크기는 `12 x 12`다.

## 14. GRU의 update gate와 reset gate가 각각 조절하는 것을 설명하라.

Update gate는 새 정보를 얼마나 반영하고 이전 상태를 얼마나 유지할지 조절한다. Reset gate는 후보 상태를 만들 때 이전 은닉상태를 얼마나 초기화하거나 반영할지 조절한다. 둘 다 정보의 반영 비율을 다루므로 0과 1 사이 값을 내는 sigmoid 계열이 적합하다.

## 15. Max Pooling과 Average Pooling의 차이를 설명하라.

Max Pooling은 윈도우 안 값 중 가장 큰 값을 대표값으로 뽑는다. 강한 특징이 있는지를 보존하는 데 유리하다. Average Pooling은 윈도우 안 값의 평균을 대표값으로 뽑아 전체적인 경향을 부드럽게 요약한다. 둘 다 공간 크기를 줄여 연산량과 메모리를 줄일 수 있지만 정보 손실이 생길 수 있다.
