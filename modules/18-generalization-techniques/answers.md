# 18. 일반화 기법 · 확인 문제 해설

## 해설 읽는 법

기본 해설은 먼저 핵심 문장을 잡고, 스스로 답할 때는 **한 문장 답 → 이유 → 예시 → 주의점** 순서로 확장한다.

## 1. 하이퍼파라미터와 파라미터의 차이를 설명하라.

파라미터는 모델이 데이터에서 직접 학습하는 값이다. 예를 들어 회귀계수나 신경망 가중치가 파라미터다. 하이퍼파라미터는 학습 전에 사람이 정하는 설정으로, 나무 깊이, 학습률, k 값 등이 있다.

## 2. 그리드 탐색과 랜덤 탐색의 차이를 설명하라.

그리드 탐색은 미리 정한 후보 조합을 빠짐없이 모두 시도한다. 랜덤 탐색은 범위 안에서 무작위로 후보를 뽑아 시도한다. 중요한 하이퍼파라미터가 일부일 때는 랜덤 탐색이 같은 예산에서 더 효율적일 수 있다.

## 3. 테스트 데이터로 하이퍼파라미터를 고르면 안 되는 이유를 설명하라.

테스트 데이터는 최종 성능을 공정하게 확인하기 위해 남겨 둔 데이터다. 이를 튜닝에 사용하면 모델 선택 과정이 테스트 데이터에 맞춰져 성능이 실제보다 좋아 보인다. 이것도 데이터 누수의 한 형태다.

## 4. 속성 선택이 과대적합을 줄일 수 있는 이유를 설명하라.

불필요한 변수나 잡음 변수가 많으면 모델이 우연한 패턴까지 배울 수 있다. 속성 선택은 중요한 변수만 남겨 모델 복잡도를 낮춘다. 그 결과 새 데이터에서 더 안정적인 성능을 얻을 수 있다.

## 5. 클래스 불균형에서 정확도가 위험한 이유를 설명하라.

다수 클래스가 대부분이면 모델이 다수 클래스만 예측해도 정확도가 높게 나온다. 하지만 중요한 소수 클래스를 놓칠 수 있다. 사기 탐지나 질병 진단에서는 재현율, 정밀도, F1 같은 지표를 함께 봐야 한다.

## 6. 오버샘플링을 훈련 데이터에만 적용해야 하는 이유를 설명하라.

검증이나 테스트 데이터에 오버샘플링을 먼저 적용하면 평가 데이터의 분포와 정보가 학습 과정에 섞일 수 있다. 그러면 실제 성능보다 좋게 보이는 누수가 생긴다. 샘플링은 교차검증의 각 훈련 fold 안에서만 해야 한다.

## 7. 왜 테스트 데이터로 하이퍼파라미터를 고르면 안 되는가?

- **한 문장 답**: 테스트 데이터 정보가 모델 선택에 새어 들어가 최종 성능 추정이 낙관적으로 변하기 때문이다.
- **이유**: 테스트 세트는 마지막에 한 번만 성능을 추정하려고 남겨 둔 데이터다. 여러 후보를 고르는 데 쓰면 테스트에 맞춘 선택이 되어 평가가 더 이상 독립적이지 않다.
- **예시**: 시험지를 보고 공부 방법을 고른 뒤 그 시험 점수를 실력이라고 말하는 것과 비슷하다.
- **주의점**: 테스트 데이터는 마지막 한 번의 확인용으로 남긴다.

## 8. 왜 오버샘플링은 훈련 데이터 안에서만 해야 하는가?

- **한 문장 답**: 검증/테스트 데이터의 정보가 합성 표본을 통해 훈련 과정에 섞일 수 있기 때문이다.
- **이유**: 합성 표본은 이웃 정보를 이용해 만들어진다. 검증/테스트 데이터가 섞이면 평가에 써야 할 정보가 훈련 데이터 형태로 새어 들어간다.
- **예시**: 전체 데이터에 SMOTE를 먼저 적용하면 검증 데이터 근처 정보가 훈련에 반영될 수 있다.
- **주의점**: 샘플링은 교차검증 fold 안에서 수행한다.

## 9. Filter, Wrapper, Embedded 속성 선택을 비교하라.

Filter는 모델 학습 전 통계량이나 관계성으로 변수를 빠르게 고른다. Wrapper는 실제 모델을 여러 번 학습해 변수 조합의 성능을 비교한다. Embedded는 모델 학습 과정 안에서 변수 선택이 함께 일어난다. 속도는 Filter가 빠르고, 계산 비용은 Wrapper가 큰 편이다.

## 10. 클래스 가중치, 임계값 조정, 오버샘플링의 차이를 설명하라.

클래스 가중치는 소수 클래스 오류에 더 큰 벌점을 주어 학습 과정에서 무시되지 않게 한다. 임계값 조정은 학습된 확률을 양성으로 판정하는 기준을 바꾼다. 오버샘플링은 훈련 데이터에서 소수 클래스 표본 수를 늘려 모델이 더 자주 보게 한다.

## 11. RFE가 어떤 순서로 속성을 제거하는지 설명하라.

RFE는 전체 속성을 사용해 모델을 먼저 학습한다. 그다음 모델이 제공하는 속성 중요도, 예를 들어 트리의 속성 중요도나 선형 모델 계수의 절대값을 보고 중요도가 낮은 속성을 제거한다. 남은 속성으로 다시 모델을 학습하고, 종료 조건을 만족할 때까지 이 과정을 반복한다.

## 12. SMOTE와 ADASYN의 합성 표본 생성 차이를 설명하라.

SMOTE는 소수 클래스 점 `x`와 그 최근접 이웃 `x_neighbor` 사이를 `x + delta(x_neighbor - x)` 형태로 보간해 합성 표본을 만든다. ADASYN도 이웃 사이 보간을 쓰지만, 주변에 다수 클래스가 많아 분류가 어려운 소수 클래스 점에 더 많은 합성 표본 수를 배정한다. 그래서 ADASYN은 어려운 경계 주변을 더 집중적으로 보강한다.

## 13. Tomek's Link가 제거하려는 데이터 포인트의 특징을 설명하라.

Tomek's Link는 서로 다른 클래스에 속한 두 점이 서로의 최근접 이웃인 경우를 말한다. 이런 점들은 결정경계 근처에서 클래스 구분을 혼동하게 만들 가능성이 높다. 언더샘플링에서는 보통 다수 클래스 쪽의 해당 점을 제거해 경계를 정리한다.

## 14. Micro Recall이 클래스 불균형 해소 효과를 판단하는 데 부적절할 수 있는 이유를 설명하라.

Micro Recall은 클래스별 표본 수를 반영해 전체적으로 집계되므로 다수 클래스 성능의 영향이 크게 남는다. 클래스 불균형이 심하면 소수 클래스 성능이 좋아졌는지 보기 어렵다. 소수 클래스 구분력이 핵심이면 Macro Recall, F1, 클래스별 Recall을 함께 봐야 한다.

## 15. Grid Search와 Random Search를 탐색 횟수와 연산량 관점에서 비교하라.

Grid Search는 후보 조합을 모두 평가하므로 조합 수가 적으면 빠짐없이 확인할 수 있지만, 하이퍼파라미터가 늘면 연산량이 크게 증가한다. Random Search는 전체 조합 중 지정한 횟수만 뽑아 평가하므로 최적 조합을 반드시 보장하지는 않지만, 같은 시간 예산에서 더 넓은 공간을 탐색할 수 있다.

## 16. 클래스별 Precision, Recall, F1, Macro Recall 공식을 쓰고 불균형 데이터에서의 의미를 설명하라.

클래스 `i`를 양성으로 놓고 보면 다음처럼 계산한다.

```text
Precision_i = TP_i / (TP_i + FP_i)
Recall_i = TP_i / (TP_i + FN_i)
F1_i = 2 * Precision_i * Recall_i / (Precision_i + Recall_i)
```

Macro Recall은 클래스별 Recall을 단순 평균한다.

```text
Macro Recall = (1 / K) * sum_i Recall_i
```

불균형 데이터에서는 다수 클래스가 전체 지표를 지배하기 쉽다. Macro Recall은 클래스별 Recall을 같은 무게로 평균하므로 소수 클래스의 놓침이 지표에 드러난다. 반대로 Micro Recall은 표본 수가 많은 클래스의 영향이 커서 소수 클래스 개선을 보기 어려울 수 있다.
