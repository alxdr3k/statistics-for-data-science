# 18. 일반화 기법

## 학습 목표

이 차시를 마치면 다음을 쉬운 말로 설명할 수 있으면 충분하다.

- 하이퍼파라미터 탐색이 모델 바깥 설정을 고르는 과정임을 이해한다.
- 속성 선택 방법을 Filter, Wrapper, Embedded로 구분한다.
- 클래스 불균형에서는 정확도보다 재현율, 정밀도, 임계값, 샘플링을 함께 봐야 함을 이해한다.

## 오늘의 한 줄

일반화 기법은 모델이 훈련 데이터가 아니라 새 데이터에서도 잘 작동하도록 조정하는 방법이다.

## 오늘 반드시 이해할 3가지

1. 하이퍼파라미터 탐색이 모델 바깥 설정을 고르는 과정임을 이해한다.
2. 속성 선택 방법을 Filter, Wrapper, Embedded로 구분한다.
3. 클래스 불균형에서는 정확도보다 재현율, 정밀도, 임계값, 샘플링을 함께 봐야 함을 이해한다.

## 이 차시 전에 알면 좋은 것

- **검증**: 튜닝을 테스트 데이터가 아닌 검증 데이터로 한다는 원칙
- **성능 지표**: 불균형에서 정확도만 보면 위험한 이유 ([처음 설명된 차시](../11-machine-learning/README.md#2-성능-지표))
- **샘플링**: 훈련 데이터 안에서만 분포를 바꿔야 하는 이유 ([처음 설명된 차시](../05-probability-distributions/README.md#1-확률변수와-분포))

## 개념 지도

```text
일반화 기법
├── 하이퍼파라미터 탐색
├── 속성 선택
├── 클래스 불균형 지표
├── 샘플링과 가중치
└── 확인 문제와 해설
```

## 학습 우선순위

- **필수**: 하이퍼파라미터와 파라미터 구분, 속성 선택의 세 방식, 불균형 데이터에서 지표와 샘플링 선택
- **심화**: SMOTE, ADASYN, Tomek's Link의 차이
- **확장**: 속성 선택을 검증 절차 안에서 수행하는 이유와 임계값 최적화

## 이 차시에서 꼭 붙잡을 설명 방식

정확도는 <a id="ref-18-클래스-불균형"></a>[클래스 불균형](#note-18-클래스-불균형)에서 쉽게 속인다. 전체의 99%가 정상인 데이터에서 전부 정상이라고 예측해도 정확도는 99%다. 하지만 우리가 찾고 싶은 사기나 질병 같은 소수 클래스는 하나도 못 찾는다. 그래서 재현율, 정밀도, F1 같은 지표가 필요하다.

## 핵심 이론

### 먼저 잡는 직관

- **하이퍼파라미터 탐색**: 모델이 스스로 배우는 값이 아니라 사람이 정하는 설정을 검증 데이터로 고르는 과정이다.
- **속성 선택**: 모든 변수를 넣기보다 예측에 도움이 되는 변수만 남기면 해석과 일반화가 좋아질 수 있다.
- **클래스 불균형 지표**: 소수 클래스가 중요한 문제에서는 정확도보다 정밀도, 재현율, F1을 더 중요하게 봐야 할 수 있다.
- **샘플링과 가중치**: 소수 클래스를 더 자주 보게 하거나 더 큰 손실을 주어 모델이 무시하지 않게 만든다.

### 1. 하이퍼파라미터 탐색

그리드 서치(Grid Search)는 후보 조합을 모두 본다. 랜덤 서치(Random Search)는 넓은 공간을 빠르게 탐색한다. 탐색은 검증 데이터나 <a id="ref-18-교차검증"></a>[교차검증](#note-18-교차검증)으로 평가해야 한다.

예를 들어 `max_depth = {3, 5, 10}`, `n_estimators = {100, 200}`이면 가능한 조합은 `(3, 100)`, `(5, 100)`, `(10, 100)`, `(3, 200)`, `(5, 200)`, `(10, 200)`이다. Grid Search는 이 모든 조합으로 검증을 수행하고 최적 성능을 보인 모델을 고른다. Random Search는 같은 후보 공간에서 지정한 횟수만큼 조합을 뽑아 검증한다.

둘의 차이는 탐색 방식, 탐색 횟수, 최적화 정확도, 연산량에서 생긴다. Grid Search는 빠짐없이 보므로 후보 공간이 작으면 명확하지만 조합 수가 늘면 연산량이 급격히 커진다. Random Search는 모든 조합을 보장하지 않지만 같은 예산에서 더 넓은 후보를 빠르게 훑을 수 있다.

![그리드 서치](assets/grid-search-493.png)

> **그림 읽기**: 하이퍼파라미터 후보 조합을 격자처럼 모두 확인하는 흐름을 본다. 넓고 촘촘할수록 비용이 커진다.

### 2. 속성 선택

Filter는 <a id="ref-18-변수"></a>[변수](#note-18-변수)와 타깃의 관계를 빠르게 점수화한다. Wrapper는 모델 성능을 보며 변수를 넣고 뺀다. Embedded는 모델 학습 과정에서 변수를 선택한다.

Filter 방식은 각 속성과 타깃 간 관계성을 통계량으로 측정한다. 정보이득(Information Gain), VIF(Variance Inflation Factor), 가설검정의 p-value나 통계량을 활용한다. 회귀 문제에서 연속형 특성과 연속형 타깃의 관계를 볼 때는 피어슨 상관도 같은 필터 기준을 쓸 수 있다.

Wrapper 방식은 속성 부분집합에 대한 모델의 검증 성능을 기준으로 선택한다. 완전탐색(Exhaustive Search)은 가능한 모든 속성 조합을 탐색하고, 전진 선택법(Forward Selection)은 하나씩 추가했을 때 성능 향상이 큰 속성을 넣는다. 후진 제거법(Backward Elimination)은 전체 속성에서 시작해 하나씩 제거했을 때 성능이 좋아지거나 덜 나빠지는 속성을 뺀다. 단계적 선택(Stepwise Selection)은 추가와 제거를 함께 고려한다.

재귀적 제거법(Recursive Feature Elimination, RFE)은 전체 속성으로 모델을 학습한 뒤, 모델이 제공하는 속성 중요도를 기준으로 중요도가 낮은 속성을 단계적으로 제거한다. 예를 들어 의사결정나무의 속성 중요도나 선형회귀 계수의 절대값을 기준으로 쓸 수 있다.

Embedded 방식은 학습 과정에서 속성이 선택된다. Lasso는 L1 규제 때문에 일부 계수가 0이 되어 해당 속성의 영향력이 사라질 수 있고, 의사결정나무는 분기 선택 과정에서 유용한 속성을 자연스럽게 고른다.

### 3. 클래스 불균형 지표

정확도보다 재현율, 정밀도, F1, 특이도, AUC 등을 본다. 어떤 오류가 더 비싼지에 따라 임계값도 조정한다.

AUC는 ROC curve 아래 면적이다. 정밀도와 재현율은 각각 “양성이라고 한 것 중 실제 양성이 얼마나 되는가”와 “실제 양성 중 얼마나 찾아냈는가”를 보므로, 불균형 데이터에서 정확도보다 더 직접적인 판단 기준이 될 수 있다.

클래스 불균형에서 특히 함께 봐야 할 지표는 F1 score와 Macro Recall이다. F1은 정밀도와 재현율을 함께 보는 지표이고, Macro Recall은 클래스별 Recall을 단순 평균해 소수 클래스의 성능 저하가 묻히지 않게 한다. 반대로 Micro Recall은 클래스별 표본 비율의 가중 평균처럼 집계되므로, 불균형 해소의 효과를 보기에는 부적절할 수 있다.

```text
Precision_i = TP_i / (TP_i + FP_i)
Recall_i = TP_i / (TP_i + FN_i)
F1_i = 2 * Precision_i * Recall_i / (Precision_i + Recall_i)

Macro Recall = (1 / K) * sum_i Recall_i
```

`TP_i`, `FP_i`, `FN_i`는 각각 클래스 `i`에 대한 True Positive, False Positive, False Negative다. Macro Recall은 각 클래스를 같은 무게로 평균하므로 소수 클래스가 묻히는 문제를 줄인다.

![클래스 불균형 대응](assets/class-imbalance-506.png)

> **그림 읽기**: 다수 클래스와 소수 클래스의 표본 수 차이가 지표를 왜곡하는 모습을 본다. 정확도만 보면 소수 클래스를 놓칠 수 있다.

### 4. 샘플링과 가중치

오버샘플링(Oversampling)은 소수 클래스를 늘리고 언더샘플링(undersampling)은 다수 클래스를 줄인다. SMOTE와 ADASYN은 합성 <a id="ref-18-표본"></a>[표본](#note-18-표본)을 만든다. 샘플링은 훈련 데이터 안에서만 해야 누수를 막을 수 있다.

클래스 가중치는 모델 학습 단계에서 소수 클래스의 가중치를 높여 소수 클래스가 더 잘 판별되도록 한다. 클래스 분류 임계점 조정은 모델이 낸 확률을 어느 기준에서 양성으로 볼지 바꾸어 소수 클래스 판정 균형을 조정한다.

Random Over Sampling은 소수 클래스에서 임의 표본을 뽑아 늘리므로 복제가 발생한다. Random Under Sampling은 다수 클래스 표본을 제거하므로 정보 손실 위험이 있다. 오버샘플링은 소수 클래스를 더 자주 보게 하지만 과적합 위험이 있고, 언더샘플링은 학습을 가볍게 만들 수 있지만 다수 클래스의 중요한 패턴을 버릴 수 있다.

![SMOTE 합성 표본](assets/smote-509.png)

> **그림 읽기**: 소수 클래스 이웃 사이를 이어 합성 표본을 만드는 방식을 본다. 단순 복제가 아니라 보간이 핵심이다.

![ADASYN 합성 표본](assets/adasyn-511.png)

> **그림 읽기**: 분류가 어려운 소수 클래스 주변에 더 많은 합성 표본을 만드는 흐름을 본다. 어려운 영역의 표본을 더 늘린다.

SMOTE(Synthetic Minority Over-Sampling Technique)는 소수 클래스의 임의 데이터 포인트 `x`를 고르고, 그 k개 최근접 이웃 중 하나 `x_neighbor`를 선택한 뒤 `x_smote = x + delta(x_neighbor - x)`, `delta ~ Uniform(0, 1)`로 사이점의 합성 표본을 만든다. 단순 복제가 아니라 이웃 사이를 보간한다.

ADASYN(Adaptive Synthetic Sampling Approach)은 주변에 다수 클래스가 많은, 즉 더 어려운 소수 클래스 표본 주변에 더 많은 합성 표본을 배정한다. 각 소수 클래스 샘플의 주변 k개 이웃 중 다수 클래스 비율 `r_i`를 계산하고, 그 비율에 따라 생성 개수 `g_i`를 나눈다. 이후 SMOTE처럼 이웃 사이 `x_adasyn = x + delta(x_neighbor - x)`를 만든다.

Tomek's Link는 언더샘플링 방법이다. 서로 다른 클래스에 속한 두 점이 서로의 최근접 이웃이면 경계에서 혼동을 일으킬 가능성이 높다고 보고, 언더샘플링 대상 클래스의 해당 데이터 포인트를 제거한다.

### 5. 속성 선택과 불균형 대응 선택

속성 선택은 Filter, Wrapper, Embedded 방식으로 구분한다. Filter는 모델을 학습하기 전에 상관, 카이제곱 검정, ANOVA F값 같은 기준으로 빠르게 걸러낸다. Wrapper는 전진 선택법이나 후진 제거법처럼 실제 모델 성능을 보며 변수 조합을 탐색한다. Embedded는 Lasso나 트리 기반 중요도처럼 모델 학습 과정 안에서 선택이 함께 일어난다.

클래스 불균형 대응은 하나만 정답이 아니다. 클래스 가중치는 소수 클래스 오류에 더 큰 손실을 주고, 임계값 조정은 모델이 낸 확률을 어디서 양성으로 자를지 바꾼다. 오버샘플링은 소수 클래스를 늘리고, 언더샘플링은 다수 클래스를 줄인다. SMOTE는 소수 클래스 이웃 사이를 보간하고, ADASYN은 더 어려운 소수 클래스 주변의 표본을 더 많이 만든다. Tomek's Link는 서로 가까운 이종 클래스 쌍을 정리해 경계 혼동을 줄이는 언더샘플링 접근이다.

평가에서는 정확도보다 혼동행렬을 먼저 확인한다. 재현율을 높이면 양성을 더 많이 잡지만 오탐이 늘 수 있고, 정밀도를 높이면 양성 판정의 신뢰도는 올라가지만 놓치는 양성이 늘 수 있다. 그래서 임계값은 지표 숫자가 아니라 업무 비용과 함께 정한다.

## 판단 기준

1. 튜닝은 검증 데이터에서 하고 테스트 데이터는 마지막 평가에만 사용한다.
2. 탐색 범위가 너무 좁거나 넓어 결과를 왜곡하지 않는지 확인한다.
3. <a id="ref-18-속성-선택"></a>[속성 선택](#note-18-속성-선택)은 교차검증 안에서 수행해 데이터 누수를 막는다.
4. 불균형 문제에서는 혼동행렬과 소수 클래스 성능을 함께 본다.
5. 오버샘플링은 훈련 데이터에만 적용한다.

## 오해와 반례

### 오해 1. 하이퍼파라미터 탐색은 테스트 데이터로 해도 된다.

테스트 데이터는 마지막 평가용이다. 탐색에 쓰면 성능 추정이 낙관적으로 변한다.

### 오해 2. 정확도가 높으면 불균형 문제는 없다.

다수 클래스만 맞혀도 정확도가 높을 수 있다. 소수 클래스 지표를 따로 봐야 한다.

### 오해 3. 오버샘플링은 전체 데이터에 먼저 적용해도 된다.

검증/테스트 정보가 훈련에 섞이는 누수가 생긴다. 훈련 데이터 안에서만 해야 한다.

## 예시 풀이

### 예시 1. 랜덤 포레스트 max_depth 고르기

여러 max_depth와 n_estimators 조합을 교차검증으로 비교해 새 데이터 성능이 좋은 설정을 고른다.

### 예시 2. 사기 탐지의 클래스 불균형

사기 거래가 1%라면 정확도보다 재현율, 정밀도, F1, AUC를 보고 클래스 가중치나 SMOTE를 검토한다.

## 오늘의 요약 5줄

1. 일반화 기법은 모델이 훈련 데이터가 아니라 새 데이터에서도 잘 작동하도록 돕는다.
2. 하이퍼파라미터는 학습 전에 정하는 설정이고 파라미터는 모델이 학습하는 값이다.
3. 그리드 서치는 모든 조합을 보고, 랜덤 서치는 지정한 횟수만큼 조합을 뽑아 본다.
4. 속성 선택은 불필요한 변수를 줄여 과대적합과 해석 부담을 낮출 수 있다.
5. 클래스 불균형에서는 정확도보다 소수 클래스의 놓침 비용을 기준으로 판단한다.

## 확인 문제

1. 하이퍼파라미터와 파라미터의 차이를 설명하라.
2. 그리드 탐색과 랜덤 탐색의 차이를 설명하라.
3. 테스트 데이터로 하이퍼파라미터를 고르면 안 되는 이유를 설명하라.
4. 속성 선택이 과대적합을 줄일 수 있는 이유를 설명하라.
5. 클래스 불균형에서 정확도가 위험한 이유를 설명하라.
6. 오버샘플링을 훈련 데이터에만 적용해야 하는 이유를 설명하라.
7. 왜 테스트 데이터로 하이퍼파라미터를 고르면 안 되는가?
8. 왜 오버샘플링은 훈련 데이터 안에서만 해야 하는가?
9. Filter, Wrapper, Embedded 속성 선택을 비교하라.
10. 클래스 가중치, 임계값 조정, 오버샘플링의 차이를 설명하라.
11. RFE가 어떤 순서로 속성을 제거하는지 설명하라.
12. SMOTE와 ADASYN의 합성 표본 생성 차이를 설명하라.
13. Tomek's Link가 제거하려는 데이터 포인트의 특징을 설명하라.
14. Micro Recall이 클래스 불균형 해소 효과를 판단하는 데 부적절할 수 있는 이유를 설명하라.
15. Grid Search와 Random Search를 탐색 횟수와 연산량 관점에서 비교하라.
16. 클래스별 Precision, Recall, F1, Macro Recall 공식을 쓰고 불균형 데이터에서의 의미를 설명하라.
