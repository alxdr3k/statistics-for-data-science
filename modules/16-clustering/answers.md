# 16. 군집화 · 확인 문제 해설

## 해설 읽는 법

기본 해설은 먼저 핵심 문장을 잡고, 스스로 답할 때는 **한 문장 답 → 이유 → 예시 → 주의점** 순서로 확장한다.

## 1. 거리 척도가 군집 결과에 중요한 이유를 설명하라.

군집화는 비슷한 점끼리 묶는 방법이므로 “비슷하다”를 거리로 정의한다. 거리 척도를 바꾸면 가까운 점과 먼 점의 판단이 달라진다. 그래서 유클리드, 맨해튼, 코사인 거리 중 데이터 의미에 맞는 것을 골라야 한다.

## 2. k-means에서 k를 정해야 하는 이유와 어려움을 설명하라.

k-means는 군집 수 k를 사람이 먼저 정해야 작동한다. 하지만 실제 데이터에 자연스러운 군집 수가 명확하지 않을 수 있다. elbow, silhouette, 업무 해석 가능성을 함께 보고 선택한다.

## 3. k-means와 k-medoids의 차이를 설명하라.

k-means는 각 군집의 평균점을 중심으로 사용한다. k-medoids는 실제 데이터 중 대표점을 중심으로 사용한다. 그래서 k-medoids는 이상치에 상대적으로 더 강할 수 있지만 계산은 더 무거울 수 있다.

## 4. GMM이 소프트 클러스터링이라고 불리는 이유를 설명하라.

GMM은 각 점을 하나의 군집에 딱 잘라 넣기보다 여러 군집에 속할 확률을 계산한다. 예를 들어 어떤 점이 A 군집 60%, B 군집 40%처럼 표현될 수 있다. 그래서 부드러운 군집화라고 부른다.

## 5. 계층적 군집화의 덴드로그램을 어떻게 해석하는지 설명하라.

덴드로그램은 점이나 군집이 어떤 순서로 합쳐지는지 나무 모양으로 보여 준다. 아래에서 가까이 합쳐지는 점들은 서로 유사하고, 높은 위치에서 합쳐지는 군집들은 더 멀다. 원하는 높이에서 자르면 군집 수를 정할 수 있다.

## 6. DBSCAN이 이상치를 찾을 수 있는 이유를 설명하라.

DBSCAN은 일정 반경 안에 충분한 이웃이 있는 밀집 지역을 군집으로 본다. 주변에 이웃이 부족한 점은 어느 군집에도 속하지 않는 noise로 남는다. 이 noise가 이상치 후보가 될 수 있다.

## 7. 왜 군집화에서는 거리 척도 선택이 중요한가?

- **한 문장 답**: 가깝다는 정의가 달라지면 같은 데이터도 다른 군집으로 묶이기 때문이다.
- **이유**: 군집화에는 정답 라벨이 없어서 가까움의 정의가 곧 묶는 기준이 된다. 유클리드, 코사인, 맨해튼 중 무엇을 쓰느냐에 따라 비슷함의 의미가 바뀐다.
- **예시**: 문서 데이터는 코사인 거리가 자연스러울 수 있고, 좌표 데이터는 유클리디언 거리가 자연스러울 수 있다.
- **주의점**: 거리 척도는 도메인 질문과 변수 타입에 맞춰 고른다.

## 8. 왜 elbow 방법만으로 k를 자동 결정했다고 말하기 어려운가?

- **한 문장 답**: 손실이 완만해지는 지점이 애매할 수 있고 업무적으로 해석 가능한 군집 수와 다를 수 있기 때문이다.
- **이유**: elbow는 손실 곡선의 모양을 보는 휴리스틱이라 꺾이는 지점이 사람마다 다르게 보일 수 있다. 군집이 업무적으로 설명 가능한지도 함께 봐야 한다.
- **예시**: 그래프상 팔꿈치가 4처럼 보여도 실제 운영에서는 3개 고객군이 더 설명 가능할 수 있다.
- **주의점**: 실루엣 점수와 도메인 해석을 함께 본다.

## 9. 코사인 거리와 마할라노비스 거리가 각각 어울리는 상황을 설명하라.

코사인 거리는 벡터의 크기보다 방향이 중요한 문서나 텍스트 벡터에 어울린다. 마할라노비스 거리는 변수들의 공분산 구조를 반영하므로 서로 관련된 수치 변수들이 있고 타원형 분포를 고려해야 할 때 유용하다. 둘 다 단순 유클리디언 거리와 다른 “가까움”을 정의한다.

## 10. 계층적 군집화의 단일, 완전, 평균 연결 기준을 비교하라.

단일 연결은 두 군집에서 가장 가까운 두 점의 거리를 군집 간 거리로 본다. 완전 연결은 가장 먼 두 점의 거리를 본다. 평균 연결은 두 군집 사이 모든 점 쌍 거리의 평균을 본다. 단일 연결은 길게 이어지는 군집이 생기기 쉽고, 완전 연결은 더 조밀한 군집을 선호한다.

## 11. 거리의 네 가지 성질을 설명하라.

거리는 음수가 아니어야 한다는 비음성성, 같은 점 사이 거리는 0이라는 동일성, `d(x, y) = d(y, x)`라는 대칭성, 직접 가는 거리가 다른 점을 거쳐 가는 거리보다 길 수 없다는 삼각부등식을 만족해야 한다. 다만 코사인 거리는 방향 비교에는 유용하지만 삼각부등식을 만족하지 않아 엄밀하게는 metric이 아니다.

## 12. k-means의 응집도(Inertia)와 k-medoids의 Total Cost가 각각 무엇을 낮추려는지 설명하라.

k-means의 Inertia는 각 점과 자신이 속한 군집 중심 `mu_k` 사이의 제곱거리 합이다. 중심점을 갱신할 때 이 값을 낮추는 방향으로 움직인다. k-medoids의 Total Cost는 각 점과 실제 데이터 대표점 `m_k` 사이 거리의 합이다. k-medoids는 군집 안에서 이 총거리를 가장 작게 만드는 실제 데이터 포인트를 대표점으로 고른다.

## 13. GMM의 책임도 `gamma_ik`가 무엇인지 설명하라.

`gamma_ik`는 데이터 `x_i`가 k번째 가우시안 군집에서 왔을 사후확률이다. 식으로는 `pi_k N(x_i | mu_k, Sigma_k)`를 모든 군집의 밀도 합으로 나눈 값이다. EM 알고리즘의 E-step에서 이 값을 계산하고, M-step에서는 `gamma_ik`를 가중치로 평균, 공분산, 혼합비를 갱신한다.

## 14. DBSCAN의 core point, border point, noise point를 구분하라.

Core point는 eps 반경 안에 `min_samples` 이상 이웃이 있어 군집을 확장할 수 있는 중심점이다. Border point는 core point의 이웃이지만 자기 주변 이웃 수는 충분하지 않은 점이다. Noise point는 어떤 core point에도 연결되지 않아 군집에 속하지 않는 점이다.

## 15. 군집 수를 사전에 정하지 않아도 되는 군집화 모델을 설명하라.

DBSCAN은 eps와 min_samples를 기준으로 밀집 영역을 연결하면서 군집 수가 수행 과정에서 정해진다. 계층적 군집화도 목표 군집 수를 직접 정할 수 있지만, 결합 임계값이나 덴드로그램 절단 높이를 기준으로 군집 수가 결정되게 할 수 있다. 반대로 k-means, k-medoids, GMM은 보통 K를 먼저 정해야 한다.

## 16. 주요 거리 척도 공식과 코사인/마할라노비스 거리의 주의점을 설명하라.

범주형 변수에는 해밍 거리와 자카드 거리를 쓸 수 있다.

```text
Hamming distance = number of positions where x_j != y_j
Jaccard distance = 1 - |A intersect B| / |A union B|
```

연속형 변수에는 유클리디언, 맨해튼, 민코프스키, 코사인, 마할라노비스 거리를 자주 비교한다.

```text
Euclidean distance = sqrt(sum_j (x_j - y_j)^2)
Manhattan distance = sum_j |x_j - y_j|
Minkowski distance = (sum_j |x_j - y_j|^p)^(1/p)
Cosine distance = 1 - (x^T y / (||x|| ||y||))
Mahalanobis distance = sqrt((x - y)^T Sigma^-1 (x - y))
```

민코프스키 거리는 `p = 2`일 때 유클리디언 거리와 같다. 코사인 거리는 방향을 비교해 스케일 영향이 작고 고차원 희소 벡터에 유용하지만, 삼각부등식을 만족하지 않아 엄밀한 metric은 아니다. 마할라노비스 거리는 공분산행렬을 사용하므로 스케일 차이뿐 아니라 변수 간 상관관계까지 반영한다.
