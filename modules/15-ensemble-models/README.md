# 15. 앙상블 모델

## 학습 목표

이 차시를 마치면 다음을 쉬운 말로 설명할 수 있으면 충분하다.

- 배깅과 부스팅의 차이를 설명한다.
- 앙상블이 분산과 편향 중 무엇을 주로 줄이는지 구분한다.
- 랜덤 포레스트가 의사결정나무와 배깅을 결합한 모델임을 이해한다.
- Hard Voting과 Soft Voting처럼 여러 모델의 예측을 결합하는 방법을 구분한다.

## 오늘의 한 줄

앙상블은 여러 약한 모델의 판단을 결합해 하나의 모델보다 안정적인 예측을 얻는 방법이다.

## 오늘 반드시 이해할 3가지

1. 배깅과 부스팅의 차이를 설명한다.
2. 랜덤 포레스트가 의사결정나무와 배깅을 결합한 모델임을 이해한다.
3. Hard Voting과 Soft Voting처럼 여러 모델의 예측을 결합하는 방법을 구분한다.

## 이 차시 전에 알면 좋은 것

- **의사결정나무**: 랜덤 포레스트와 부스팅의 기본 학습기
- **과대적합**: 여러 모델을 합쳐 흔들림을 줄이는 이유
- **검증**: 앙상블도 새 데이터 성능으로 평가해야 한다

## 개념 지도

```text
앙상블 모델
├── 배깅
├── 랜덤 포레스트
├── 부스팅
├── AdaBoost
├── Gradient Boosting
├── Hard/Soft Voting
└── 확인 문제와 해설
```

## 학습 우선순위

- **필수**: 배깅은 분산 감소에 강함, 부스팅은 순차적 오류 보완, 랜덤 포레스트는 데이터와 변수 무작위성을 함께 사용
- **심화**: AdaBoost와 Gradient Boosting의 보완 신호 차이, OOB 샘플의 의미
- **확장**: 손실함수와 결합 가중치가 부스팅 학습에 들어가는 방식

## 이 차시에서 꼭 붙잡을 설명 방식

<a id="ref-15-앙상블"></a>[앙상블](#note-15-앙상블)이 항상 좋아지는 것은 아니다. 서로 똑같이 틀리는 모델을 많이 모아도 오류가 줄지 않는다. 효과를 얻으려면 모델들이 어느 정도 잘 맞히면서도 서로 다른 방식으로 틀려야 한다. 다양성이 핵심이다.

초심자는 앙상블을 편향과 분산 관점으로 나눠 읽으면 쉽다. 배깅은 비슷한 모델 여러 개의 흔들림을 평균내므로 분산 감소에 강하다. 부스팅은 앞 모델이 못 잡은 패턴을 다음 모델이 보완하므로 편향 감소에 도움이 될 수 있다. 다만 부스팅도 너무 오래 반복하면 훈련 데이터의 잡음까지 따라가 과대적합될 수 있다.

## 핵심 이론

### 먼저 잡는 직관

- **배깅**: 여러 데이터 샘플로 비슷한 모델을 따로 학습시켜 평균이나 투표로 흔들림을 줄인다.
- **랜덤 포레스트**: 나무마다 데이터와 변수 선택을 다르게 해 서로 다른 판단을 만든 뒤 투표한다.
- **부스팅**: 앞 모델이 틀린 부분에 다음 모델이 더 집중하면서 순서대로 약점을 보완한다.
- **보팅**: 서로 다른 기반 모델의 예측을 다수결이나 확률 평균으로 결합한다.

앙상블의 일반적인 예측식은 `F_M(x) = sum_{m=1}^M gamma_m f_m(x)`로 쓸 수 있다. `M`은 모델 개수, `f_m(x)`는 m번째 기반 모델의 예측, `gamma_m`은 그 모델에 주는 가중치다. 모든 모델을 똑같이 믿는다면 `gamma_m = 1 / M`처럼 두고 평균을 내며, 모델별 신뢰도가 다르면 가중치를 다르게 줄 수 있다.

### 1. 배깅

배깅(Bagging)은 Bootstrap과 Aggregation의 결합이다. <a id="ref-15-부트스트랩"></a>[부트스트랩](#note-15-부트스트랩) <a id="ref-15-표본"></a>[표본](#note-15-표본)을 여러 개 만들고 각 표본으로 모델을 학습한 뒤 평균이나 투표로 합친다. 분산을 줄이는 데 강하다.

Bootstrap은 전체 데이터에서 복원추출(with replacement)로 원 데이터와 같은 크기의 학습셋을 여러 번 만드는 방법이다. 한 번도 뽑히지 않는 데이터가 생기는데, 전체 데이터 수가 충분히 크면 그 비율은 `(1 - 1/n)^n ≈ e^-1 ≈ 0.368`, 즉 약 36.8%다. 이 데이터가 Out-of-Bag(OOB) 샘플이고, 해당 부트스트랩 모델이 학습에 쓰지 않은 데이터이므로 모델 검증에 활용할 수 있다.

Aggregation은 여러 모델 예측을 집계하는 단계다. 회귀에서는 보통 `F_M(x) = 1/M sum_m f_m(x)`처럼 평균을 내고, 분류에서는 투표를 쓴다. 배깅이 분산을 줄이는 이유는 각 모델이 조금씩 다른 부트스트랩 표본에서 학습해 흔들림이 서로 상쇄되기 때문이다.

평균으로 분산이 줄어드는 직관은 표본평균과 비슷하다. 서로 완전히 같은 모델을 평균내면 달라지는 것이 거의 없지만, 서로 조금씩 다르게 흔들리는 모델을 평균내면 우연한 오차가 일부 상쇄된다. 그래서 랜덤 포레스트는 데이터 표본뿐 아니라 분기 후보 변수도 무작위로 바꿔 나무들 사이의 차이를 만든다.

### 2. 랜덤 포레스트

랜덤 포레스트(Random Forest)는 <a id="ref-15-의사결정나무"></a>[의사결정나무](#note-15-의사결정나무)를 많이 만들되 표본과 <a id="ref-15-변수"></a>[변수](#note-15-변수) 선택에 무작위성을 넣는다. 나무들이 서로 달라져 앙상블 효과가 커진다.

의사결정나무는 데이터가 조금 바뀌어도 구조가 크게 달라질 수 있다. 랜덤 포레스트는 이 민감함을 오히려 활용해 부트스트랩으로 데이터 구성을 바꾸고, 각 분기에서 후보 속성을 일부만 보게 한다. 이렇게 데이터와 속성 양쪽에 다양성을 넣으면 여러 강한 나무가 서로 다른 가설 공간을 보고 예측하게 되어, 단일 나무보다 안정적인 강한 학습기를 만들 수 있다.

![랜덤 포레스트](assets/random-forest-410.png)

> **그림 읽기**: 여러 트리에 데이터와 변수 무작위성을 넣는 구조를 본다. 서로 다른 트리를 만들어 앙상블 효과를 높인다.

### 3. 부스팅

앞 모델이 틀린 부분을 다음 모델이 더 신경 쓰게 만들어 순차적으로 오류를 줄인다. 성능은 강하지만 과적합 관리가 중요하다. 일반 형태는 `F_m(x) = F_{m-1}(x) + learning rate * gamma_m f_m(x)`처럼 이전 모델에 새 기반 모델을 더하는 방식이다. 새 모델과 가중치 `gamma_m`은 손실함수 `L`이 줄어드는 방향으로 정한다.

```text
F_m(x) = F_{m-1}(x)
         + argmin_{gamma_m, f_m} sum_i L(y_i, F_{m-1}(x_i) + gamma_m f_m(x_i))
```

실제 구현에서는 새 모델을 더할 때 학습률을 곱해 한 번에 너무 크게 움직이지 않게 한다.

AdaBoost는 틀린 데이터의 가중치를 키워 다음 모델이 그 데이터를 더 신경 쓰게 만든다. 경사 부스팅은 현재 모델이 남긴 잔차나 손실이 줄어드는 방향을 다음 모델이 맞추는 방식이다. 둘 다 “약한 모델을 순서대로 보완한다”는 큰 틀은 같지만, 무엇을 보완 신호로 삼는지가 다르다.

![AdaBoost의 가중치 갱신](assets/adaboost-412.png)

> **그림 읽기**: 틀린 데이터의 가중치를 키워 다음 모델이 더 신경 쓰게 하는 흐름을 본다. 순차적 보완이 boost의 핵심이다.

![Gradient Boosting의 잔차 학습](assets/gradient-boosting-417.png)

### 4. 여러 종류의 예측기 결합

앙상블은 같은 종류의 모델만 결합하지 않는다. 예를 들어 `f_1(x)`는 선형회귀, `f_2(x)`는 경사 부스팅처럼 서로 다른 기반 모델을 둘 수 있다. 최종 예측은 `F_M(x) = sum_m gamma_m f_m(x)`처럼 가중합으로 표현하며, 가중치를 모두 `1/M`으로 두거나 모델별로 다르게 지정할 수 있다.

### 5. Hard/Soft Voting과 Gradient Boosting

보팅은 hard voting과 soft voting으로 나눈다. Hard voting은 각 모델이 고른 클래스에 투표해 다수결로 정한다. Soft voting은 각 모델이 낸 클래스 확률을 평균한 뒤 가장 높은 확률의 클래스를 고른다. 확률이 잘 보정된 모델들이라면 soft voting이 더 많은 정보를 쓸 수 있다.

AdaBoost는 틀린 샘플의 가중치를 키우고, 다음 약한 학습기가 그 샘플을 더 신경 쓰도록 만든다. 처음에는 데이터 가중치 `w_i`의 합이 1이 되도록 초기화하고 `F_0 = 0`에서 시작한다. m번째 기반 모델 `f_m`을 학습한 뒤 손실을 최소화하는 `gamma_m`을 구하고, `F_m(x) = F_{m-1}(x) + learning rate * gamma_m f_m(x)`로 병합한다. 그다음 새 손실을 기준으로 `w`를 갱신하고 합이 1이 되도록 정규화한다.

학습률은 부스팅에서 한 모델의 보완을 얼마나 크게 반영할지 정한다. 학습률이 작으면 천천히 보완하므로 보통 더 많은 기반 모델이 필요하고, 학습률이 크면 빠르게 맞추지만 과대적합 위험이 커질 수 있다.

AdaBoost에서 지수손실 `L(y_i, F_m(x_i)) = e^{-y_i F_m(x_i)}`를 쓰고 `y_i in {-1, 1}`일 때, m번째 모델의 가중 오류율을 `epsilon_m`이라고 두면 `gamma_m = 1/2 log((1 - epsilon_m) / epsilon_m)`로 계산한다. 많이 틀린 모델은 작은 가중치를 받고, 잘 맞힌 모델은 큰 가중치를 받는다.

Gradient Boosting은 손실함수를 직접 보고, 현재 모델이 줄여야 할 방향을 다음 모델이 맞춘다. 먼저 `F_0`은 전체 손실을 가장 작게 하는 상수 예측값으로 시작한다. 이후 `g_m = -nabla_{F_{m-1}(x)} L(y, F_{m-1}(x))`처럼 음의 경사를 만들고, 이 값을 대상 변수처럼 두어 다음 기반 모델 `f_m(x)`을 학습한다. 회귀의 제곱손실 `L(y, F(x)) = 1/2(y - F(x))^2`에서는 이 음의 경사가 잔차 `y - F_{m-1}(x)`가 된다.

제곱손실에서는 새 모델을 얼마나 더할지 정하는 `gamma_m`도 닫힌 형태로 구할 수 있다. 직관적으로는 새 기반 모델 `f_m(x_i)`가 현재 잔차 `y_i - F_{m-1}(x_i)`를 얼마나 잘 설명하는지 보는 값이고, 식으로는 `sum_i f_m(x_i)(y_i - F_{m-1}(x_i)) / sum_i f_m(x_i)^2`다. 닫힌 형태로 구하기 어려운 손실이면 line search 같은 최적화 알고리즘으로 찾는다.

```text
L(y, F_m(x)) = 1/2(y - F_m(x))^2
negative gradient = y - F_{m-1}(x)

gamma_m =
sum_i f_m(x_i)(y_i - F_{m-1}(x_i))
/ sum_i f_m(x_i)^2
```

## 판단 기준

1. 여러 모델이 서로 충분히 다른 실수를 하는지 확인한다.
2. <a id="ref-15-배깅"></a>[배깅](#note-15-배깅)은 분산 감소, <a id="ref-15-부스팅"></a>[부스팅](#note-15-부스팅)은 편향 감소에 더 초점이 있음을 구분한다.
3. <a id="ref-15-랜덤-포레스트"></a>[랜덤 포레스트](#note-15-랜덤-포레스트)의 나무 수, 깊이, 변수 샘플링 설정을 확인한다.
4. 부스팅은 학습률과 반복 수가 과대적합에 미치는 영향을 본다.
5. Voting은 hard voting인지 soft voting인지, 그리고 확률 평균에 가중치를 쓰는지 확인한다.
6. OOB 샘플을 쓰는 배깅 검증인지 별도 검증셋을 쓰는지 구분한다.

## 오해와 반례

### 오해 1. 모델을 많이 합치면 항상 좋아진다.

서로 비슷하게 틀리는 모델을 많이 모으면 효과가 작다. 다양성이 필요하다.

### 오해 2. 배깅과 부스팅은 같은 방식이다.

배깅은 병렬적으로 분산을 줄이고, 부스팅은 순차적으로 오류를 보완한다.

### 오해 3. 랜덤 포레스트는 순차적으로 학습되므로 병렬화가 어렵다.

랜덤 포레스트는 의사결정나무를 기반 모델로 쓰는 배깅 모델이다. 각 나무는 독립적으로 만들 수 있으므로 모델 단위 병렬화가 비교적 쉽다.

앞 모델이 틀린 데이터에 다음 모델이 더 신경 쓰면 점차 약점을 보완할 수 있다.

## 오늘의 요약 5줄

1. 앙상블은 여러 모델의 판단을 결합해 더 안정적인 예측을 얻는 방법이다.
2. 배깅은 데이터를 다르게 뽑아 병렬 모델을 만들고 평균으로 흔들림을 줄인다.
3. 랜덤 포레스트는 배깅에 변수 무작위성을 더한 의사결정나무 앙상블이다.
4. 부스팅은 이전 모델의 오류를 다음 모델이 보완하도록 순차적으로 학습한다.
5. 앙상블도 다양성이 부족하거나 검증 설계가 잘못되면 성능이 좋아지지 않는다.

## 확인 문제

1. 앙상블이 성능을 높일 수 있는 이유를 설명하라.
2. 배깅과 부스팅의 차이를 설명하라.
3. 랜덤 포레스트에서 변수를 무작위로 고르는 이유를 설명하라.
4. 부스팅이 과대적합될 수 있는 이유를 설명하라.
5. Hard Voting과 Soft Voting의 차이를 설명하라.
6. 앙상블 모델의 해석이 어려워질 수 있는 이유를 설명하라.
7. 왜 배깅은 분산을 줄이는 데 도움이 되는가?
8. 왜 boosting은 “boost”라는 이름처럼 약한 모델을 강하게 만들 수 있는가?
9. OOB 샘플이 무엇이며 왜 검증에 활용할 수 있는가?
10. AdaBoost에서 `gamma_m = 1/2 log((1 - epsilon_m) / epsilon_m)`가 의미하는 바를 설명하라.
11. Gradient Boosting이 잔차 또는 손실의 기울기를 학습한다는 뜻을 설명하라.
12. 랜덤 포레스트가 모델 단위 병렬화에 유리한 이유를 설명하라.
13. 과적합 상태의 Gradient Boosting에서 기반 모델 수를 늘리는 조치가 부적절할 수 있는 이유를 설명하라.
14. 일반 부스팅 목적식과 제곱손실 Gradient Boosting의 `gamma_m` 공식을 쓰라.
15. 배깅이 분산 감소에 강한 이유를 표본평균 직관으로 설명하라.
16. 랜덤 포레스트가 변수 무작위성을 추가하는 이유를 모델 다양성 관점에서 설명하라.
17. 부스팅에서 학습률과 기반 모델 수가 함께 조절되어야 하는 이유를 설명하라.
