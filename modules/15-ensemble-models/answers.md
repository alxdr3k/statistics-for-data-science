# 15. 앙상블 모델 · 확인 문제 해설

## 1. 앙상블이 성능을 높일 수 있는 이유를 설명하라.

여러 모델이 서로 다른 실수를 한다면 평균이나 투표로 실수가 일부 상쇄된다. 한 모델의 판단에만 의존할 때보다 예측이 안정적일 수 있다. 핵심은 모델들이 충분히 다양해야 한다는 점이다.

## 2. 배깅과 부스팅의 차이를 설명하라.

배깅은 여러 모델을 병렬로 학습시키고 결과를 평균내거나 투표한다. 부스팅은 앞 모델이 틀린 부분을 다음 모델이 더 집중해 순서대로 보완한다. 배깅은 분산 감소, 부스팅은 편향 감소에 더 초점이 있다.

## 3. 랜덤 포레스트에서 변수를 무작위로 고르는 이유를 설명하라.

모든 나무가 같은 강한 변수만 보고 비슷한 구조가 되면 앙상블의 다양성이 줄어든다. 각 분기에서 후보 변수를 일부만 보게 하면 나무들이 서로 다른 판단을 만들 가능성이 커진다. 이 다양성이 평균이나 투표의 효과를 높인다.

## 4. 부스팅이 과대적합될 수 있는 이유를 설명하라.

부스팅은 이전 모델의 오류를 계속 줄이려 하므로 반복이 너무 많거나 학습률이 부적절하면 잡음까지 맞추려 할 수 있다. 그러면 훈련 성능은 좋아져도 검증 성능은 나빠진다. 조기 종료와 검증 성능 확인이 필요하다.

## 5. 보팅과 스태킹의 차이를 설명하라.

보팅은 여러 모델의 예측을 다수결이나 평균으로 단순 결합한다. 스태킹은 여러 모델의 예측값을 새 입력으로 삼아 또 다른 모델이 최종 판단을 학습한다. 스태킹은 강력할 수 있지만 데이터 누수를 더 조심해야 한다.

## 6. 앙상블 모델의 해석이 어려워질 수 있는 이유를 설명하라.

여러 모델의 판단을 결합하면 최종 예측이 어떤 단일 규칙에서 나왔는지 보기 어렵다. 특히 부스팅이나 스태킹은 내부 모델이 많고 상호작용도 복잡하다. 그래서 변수 중요도나 SHAP 같은 보조 해석 도구가 필요할 수 있다.
