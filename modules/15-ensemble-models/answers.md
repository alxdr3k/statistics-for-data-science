# 15. 앙상블 모델 · 확인 문제 해설

## 해설 읽는 법

기본 해설은 먼저 핵심 문장을 잡고, 스스로 답할 때는 **한 문장 답 → 이유 → 예시 → 주의점** 순서로 확장한다.

## 1. 앙상블이 성능을 높일 수 있는 이유를 설명하라.

여러 모델이 서로 다른 실수를 한다면 평균이나 투표로 실수가 일부 상쇄된다. 한 모델의 판단에만 의존할 때보다 예측이 안정적일 수 있다. 핵심은 모델들이 충분히 다양해야 한다는 점이다.

## 2. 배깅과 부스팅의 차이를 설명하라.

배깅은 여러 모델을 병렬로 학습시키고 결과를 평균내거나 투표한다. 부스팅은 앞 모델이 틀린 부분을 다음 모델이 더 집중해 순서대로 보완한다. 배깅은 분산 감소, 부스팅은 편향 감소에 더 초점이 있다.

## 3. 랜덤 포레스트에서 변수를 무작위로 고르는 이유를 설명하라.

모든 나무가 같은 강한 변수만 보고 비슷한 구조가 되면 앙상블의 다양성이 줄어든다. 각 분기에서 후보 변수를 일부만 보게 하면 나무들이 서로 다른 판단을 만들 가능성이 커진다. 이 다양성이 평균이나 투표의 효과를 높인다.

## 4. 부스팅이 과대적합될 수 있는 이유를 설명하라.

부스팅은 이전 모델의 오류를 계속 줄이려 하므로 반복이 너무 많거나 학습률이 부적절하면 잡음까지 맞추려 할 수 있다. 그러면 훈련 성능은 좋아져도 검증 성능은 나빠진다. 조기 종료와 검증 성능 확인이 필요하다.

## 5. Hard Voting과 Soft Voting의 차이를 설명하라.

Hard Voting은 각 모델이 고른 클래스에 한 표씩 투표해 다수결로 최종 클래스를 정한다. Soft Voting은 각 모델이 낸 클래스 확률을 평균하거나 가중평균한 뒤 가장 높은 확률의 클래스를 고른다.

## 6. 앙상블 모델의 해석이 어려워질 수 있는 이유를 설명하라.

여러 모델의 판단을 결합하면 최종 예측이 어떤 단일 규칙에서 나왔는지 보기 어렵다. 특히 랜덤 포레스트나 부스팅은 내부 모델이 많고 상호작용도 복잡하다. 그래서 개별 모델 하나보다 전체 판단 과정을 해석하기 어려울 수 있다.

## 7. 왜 배깅은 분산을 줄이는 데 도움이 되는가?

- **한 문장 답**: 서로 조금씩 다른 모델의 예측을 평균내면 개별 모델의 흔들림이 상쇄되기 때문이다.
- **이유**: 각 모델이 다른 부트스트랩 표본에서 학습하면 오차가 조금씩 다르게 생긴다. 평균을 내면 방향이 다른 흔들림은 줄어든다.
- **예시**: 여러 사람이 독립적으로 추정한 값을 평균내면 한 사람의 실수가 덜 크게 작용한다.
- **주의점**: 모델들이 모두 똑같이 틀리면 앙상블 효과는 작다.

## 8. 왜 boosting은 “boost”라는 이름처럼 약한 모델을 강하게 만들 수 있는가?

- **한 문장 답**: 앞 모델이 못 맞힌 부분을 다음 모델이 더 집중해서 보완하기 때문이다.
- **이유**: 약한 모델 하나는 부족해도 다음 모델이 남은 오류에 집중하면 전체 예측은 단계적으로 보완된다. 순차 결합이 단일 약한 모델보다 표현력을 키운다.
- **예시**: 처음 나무가 놓친 고객군에 가중치를 더 주고 다음 나무가 그 부분을 학습한다.
- **주의점**: 순차적으로 보완하므로 반복 수와 학습률을 조절하지 않으면 과대적합될 수 있다.

## 9. OOB 샘플이 무엇이며 왜 검증에 활용할 수 있는가?

OOB(Out-of-Bag) 샘플은 부트스트랩 복원추출 과정에서 특정 기반 모델의 학습 데이터에 한 번도 뽑히지 않은 데이터다. 원 데이터와 같은 크기로 복원추출하면 충분히 큰 n에서 약 `(1 - 1/n)^n ≈ e^-1 ≈ 0.368`, 즉 36.8%가 OOB로 남는다. 해당 모델 입장에서는 학습에 쓰지 않은 데이터이므로 일반화 성능 검증에 활용할 수 있다.

## 10. AdaBoost에서 `gamma_m = 1/2 log((1 - epsilon_m) / epsilon_m)`가 의미하는 바를 설명하라.

`epsilon_m`은 m번째 기반 모델의 가중 오류율이다. 오류율이 낮으면 `(1 - epsilon_m) / epsilon_m`가 커져 `gamma_m`도 커지고, 그 모델의 최종 예측 영향력이 커진다. 반대로 오류율이 높으면 `gamma_m`이 작아져 덜 믿게 된다. 그래서 AdaBoost는 틀린 데이터의 가중치를 조정하면서 잘 맞히는 약한 모델에 더 큰 결합 가중치를 준다.

## 11. Gradient Boosting이 잔차 또는 손실의 기울기를 학습한다는 뜻을 설명하라.

Gradient Boosting은 현재 모델이 틀린 방향을 다음 약한 모델이 보완하게 한다. 제곱오차 회귀에서는 남은 잔차를 맞추는 것처럼 보이고, 일반 손실에서는 손실을 가장 빨리 줄이는 음의 기울기를 맞추는 것으로 이해할 수 있다. 그래서 모델을 하나씩 더하며 손실을 줄인다.

## 12. 랜덤 포레스트가 모델 단위 병렬화에 유리한 이유를 설명하라.

랜덤 포레스트는 의사결정나무를 기반 모델로 쓰는 배깅 모델이다. 배깅의 각 기반 모델은 서로 다른 부트스트랩 표본과 변수 후보를 보고 독립적으로 학습할 수 있다. 앞 나무의 결과가 다음 나무의 입력이 되는 구조가 아니므로 여러 나무를 동시에 만들기 쉽다.

## 13. 과적합 상태의 Gradient Boosting에서 기반 모델 수를 늘리는 조치가 부적절할 수 있는 이유를 설명하라.

Gradient Boosting은 모델을 하나씩 더하면서 현재 손실을 계속 줄인다. 이미 과적합 상태라면 기반 모델을 더 추가하는 것이 훈련 데이터의 잡음까지 더 맞추는 방향으로 갈 수 있다. 이런 상황에서는 모델 수를 늘리기보다 학습률을 낮추거나, 조기 종료를 쓰거나, 나무 깊이 같은 복잡도를 줄이는 쪽을 먼저 검토한다.

## 14. 일반 부스팅 목적식과 제곱손실 Gradient Boosting의 `gamma_m` 공식을 쓰라.

일반 부스팅은 이전 모델에 새 기반 모델을 더해 손실을 줄이는 방향으로 간다.

```text
F_m(x) = F_{m-1}(x)
         + argmin_{gamma_m, f_m} sum_i L(y_i, F_{m-1}(x_i) + gamma_m f_m(x_i))
```

제곱손실에서는 음의 경사가 잔차가 된다.

```text
L(y, F_m(x)) = 1/2(y - F_m(x))^2
negative gradient = y - F_{m-1}(x)
```

새 기반 모델 `f_m`을 얼마나 더할지는 다음처럼 계산한다.

```text
gamma_m =
sum_i f_m(x_i)(y_i - F_{m-1}(x_i))
/ sum_i f_m(x_i)^2
```

닫힌형으로 계산하기 어려운 손실에서는 line search 같은 최적화 방법으로 `gamma_m`을 찾을 수 있다.
