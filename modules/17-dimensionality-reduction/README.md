## 이 차시에서 꼭 붙잡을 설명 방식

<a id="ref-17-pca"></a>[PCA](#note-17-pca)는 “<a id="ref-17-변수"></a>[변수](#note-17-변수)를 그냥 버리는 것”이 아니다. 데이터가 가장 많이 퍼진 방향을 새 축으로 만들고, 그 축을 앞에서부터 사용한다. 그래서 원래 변수 여러 개가 섞인 새 변수, 즉 주성분이 생긴다.

PCA를 하기 전에는 표준화 여부를 먼저 판단한다. 원 단위가 큰 변수가 분산도 크게 가지면, PCA는 그 변수를 정보가 많은 변수로 볼 수 있다. 키는 cm, 소득은 원 단위처럼 스케일이 크게 다르면 소득이 주성분을 지배할 수 있다. 변수 단위가 다르면 보통 표준화 후 PCA를 고려한다.

## 핵심 이론

### 먼저 잡는 직관

- **차원 축소의 목적**: 차원의 저주를 완화해 성능을 높이고, 노이즈를 줄이고, 계산을 효율화하고, 시각화하기 위해 쓴다.
- **PCA**: PCA는 데이터가 가장 많이 퍼진 방향을 새 축으로 삼아 정보를 많이 남기며 차원을 줄인다.
- **SVD**: SVD는 행렬을 중요한 방향과 크기로 분해해 PCA와 추천, 압축의 기반이 된다.
- **MDS와 t-SNE**: MDS와 t-SNE는 점들 사이의 거리나 이웃 관계가 낮은 차원에서도 비슷하게 보이도록 배치한다.
- **LDA와 PLS**: LDA는 클래스 분리를 잘 보이게 하고, PLS는 입력과 출력의 공분산을 잘 설명하는 축을 찾는다.

### 1. PCA

공분산 구조를 이용해 분산을 많이 설명하는 축을 찾는다. 고유값은 각 주성분이 설명하는 분산의 크기이고, 고유벡터는 방향이다.

PCA의 관점에서 분산은 한 변수가 설명하는 정보량이고, 공분산은 두 변수가 공통으로 지닌 정보량이다. PCA는 변수 간 공분산이 0이 되도록 선형변환을 유도한다. 공분산이 0이라는 말은 새 성분끼리 중복되는 정보가 없다는 뜻이다.

주성분은 원래 변수의 이름을 그대로 갖지 않는다. 예를 들어 `PC1 = 0.7 * 수학 + 0.6 * 과학 - 0.2 * 결석일`처럼 여러 변수가 섞인 새 축일 수 있다. 그래서 PCA 결과를 해석할 때는 각 원래 변수가 주성분에 얼마나 섞였는지, 즉 loading을 함께 봐야 한다.

성분(component)은 원래 변수를 선형변환해 만든 새 축이다. 성분의 분산은 그 성분이 설명하는 정보량으로 읽는다. PCA는 각 성분의 분산 합과 원래 변수의 분산 합이 같도록, 즉 전체 정보량을 같은 총량으로 옮겨 놓도록 유도한다. 다만 앞의 몇 개 성분만 쓰면 뒤 성분의 정보는 버리므로 정보 손실이 생긴다.

계산은 먼저 데이터 행렬 `X`를 중심평균화해 `X_c = X - Xbar`를 만드는 것에서 시작한다. 각 열에 평균을 빼도 분산 정보는 바뀌지 않고, 공분산행렬은 `Cov(X_c) = X_c^T X_c / (n - 1)`로 쓸 수 있다. 새 좌표를 `Z = X_c W`라고 두면 `Cov(Z) = W^T Cov(X_c) W`가 된다. 이 공분산이 대각행렬 `Lambda`가 되도록 하는 `W`를 찾는 것이 PCA다.

공분산행렬은 대칭행렬이므로 고유벡터를 정규직교(orthonormal)하게 잡을 수 있다. 그래서 PCA는 결국 `A W = W Lambda`라는 공분산행렬의 고윳값분해 문제다. `Lambda`의 대각 원소인 고윳값은 각 주성분의 분산이고, 고윳값 비율은 각 성분이 전체 분산 중 얼마를 설명하는지 나타낸다. 고유벡터의 각 원소는 원래 변수가 해당 주성분에 얼마나 섞이는지 보여 주며, 영향도는 보통 절대값이 클수록 크다고 해석한다.

trace는 행렬 대각합이다. `trace(W^-1 A W) = trace(A)`가 되므로 고윳값의 합, 즉 성분 분산의 합은 원래 공분산행렬의 대각합과 같다. 이것이 “전체 분산 총량은 보존하고, 그중 앞 성분을 선택해 차원을 줄인다”는 PCA 해석의 수식적 이유다.

![PCA의 직관](assets/pca-intuition-456.png)

> **그림 읽기**: 데이터가 가장 많이 퍼진 방향을 새 축으로 잡는 모습을 본다. 정보 보존은 설명분산으로 판단한다.

### 2. SVD

SVD는 행렬을 직교 방향, 특이값, 또 다른 직교 방향의 세 부분으로 분해해 중요한 구조를 추출한다. PCA와 연결되며 큰 데이터의 저차원 표현에 자주 쓰인다.

특이값 분해(Singular Value Decomposition)는 `m x n` 행렬 `M`을 `M = U Sigma V^T`로 분해한다. `U`는 좌측 특이행렬, `Sigma`는 대각 원소가 0 이상인 특이값행렬, `V`는 우측 특이행렬이다. `U`와 `V`는 정규직교 행렬이라 `U^-1 = U^T`, `V^-1 = V^T`로 다룰 수 있다.

`M M^T = U Sigma Sigma^T U^T`이므로 `U`는 `M M^T`의 고유벡터와 연결되고, `M^T M = V Sigma^T Sigma V^T`이므로 `V`는 `M^T M`의 고유벡터와 연결된다. PCA와 연결할 때는 중심평균화한 데이터 `X_c`에 대해 `X_c = U Sigma V^T`라고 두면, 공분산행렬 `X_c^T X_c / (n - 1) = V (Sigma^T Sigma / (n - 1)) V^T`가 된다. 따라서 PCA의 고유벡터는 `V`, 고윳값은 `Sigma^T Sigma / (n - 1)`와 연결된다.

SVD는 Full SVD, Reduced(Thin) SVD, Truncated SVD로 구분할 수 있다. Full SVD는 전체 직교 기저를 모두 들고 가고, Reduced SVD는 실제 rank에 필요한 얇은 행렬만 남기며, Truncated SVD는 큰 특이값 일부만 남겨 압축과 차원 축소에 쓴다.

![특이값 분해](assets/svd-decomposition-462.png)

> **그림 읽기**: 행렬을 방향과 크기의 곱으로 분해하는 구조를 본다. 큰 특이값이 중요한 구조를 나타낸다.

### 3. MDS와 t-SNE

MDS는 거리 관계를 낮은 차원에 보존하려 하고, t-SNE는 가까운 이웃 구조를 시각화하는 데 강하다. t-SNE는 새 데이터 변환과 재현성에 주의해야 한다.

다차원 척도법(MDS)은 데이터 포인트 사이의 유사도 또는 거리를 유지하면서 낮은 차원 좌표를 찾는다. 고전적 MDS는 유클리드 거리행렬에서 내적행렬을 복원한다. 중심화행렬 `H = I - 11^T / n`을 쓰면 거리제곱행렬 `D^2`에서 `X_c X_c^T = -1/2 H D^2 H`를 얻을 수 있고, 이를 고윳값분해해 좌표를 만든다. 유클리드 거리행렬만 주어진 경우에도 쓸 수 있으며, 원 데이터가 있을 때는 PCA 변환값과 동치로 해석할 수 있다.

Metric MDS와 Non-metric MDS는 모두 낮은 차원에서 Stress 함수를 줄이는 방식이다. Metric MDS는 원래 거리 `d_ij`와 축소 공간 거리 `delta_ij`의 차이 `sum_{i<j}(d_ij - delta_ij)^2`를 줄인다. Non-metric MDS는 거리의 절대값보다 거리 순위 관계를 보존하는 데 초점을 둔다.

![t-SNE의 이웃 확률](assets/tsne-intuition-472.png)

> **그림 읽기**: 원래 공간의 가까운 이웃 관계를 낮은 차원에서도 가깝게 보이게 하는 흐름을 본다. 축 자체보다 이웃 구조가 중요하다.

### 4. LDA와 PLS

LDA는 클래스 구분이 잘 되는 축을 찾고, PLS는 X와 y의 공분산을 잘 설명하는 성분을 찾는다. 지도 정보가 들어간다는 점이 PCA와 다르다.

선형판별분석(LDA)은 클래스 간 분산은 크게, 클래스 내부 분산은 작게 유지하는 차원 축소다. 클래스 간 산포행렬은 `S_B = sum_k N_k (mu_k - mu)(mu_k - mu)^T`, 클래스 내부 산포행렬은 `S_W = sum_k sum_{x in C_k} (x - mu_k)(x - mu_k)^T`로 본다. Fisher criterion은 `J(w) = w^T S_B w / w^T S_W w`이고, 변환 벡터는 `S_B W = S_W W Lambda`라는 일반화된 고윳값 문제로 구한다. 얻을 수 있는 선형판별벡터 수는 최대 `min(K - 1, W의 차원)`개다.

![LDA의 Fisher Criterion](assets/lda-fisher-478.png)

> **그림 읽기**: 클래스 사이 거리는 크게, 클래스 안 퍼짐은 작게 만드는 방향을 본다. PCA와 달리 라벨 정보를 사용한다.

### 5. MDS, t-SNE, PLS를 구분하기

MDS는 점 사이의 거리표를 낮은 차원 좌표로 바꾸는 방법이다. 고전적 MDS는 거리 정보를 내적 구조로 바꾸고, 그 구조를 분해해 좌표를 만든다. Metric MDS는 거리의 크기 자체를 맞추려 하고, Non-metric MDS는 거리의 순서 관계를 보존하는 데 더 초점을 둔다.

t-SNE는 모든 거리를 똑같이 보존하려 하지 않는다. 원래 공간에서는 데이터 포인트 `i`, `j`가 이웃일 확률 `p_ij`를 가우시안 분포 기반으로 계산한다. 이때 각 점의 `sigma_i^2`는 perplexity 값을 기준으로 자동으로 조정된다. 축소 공간에서는 `q_ij`를 자유도 1인 Student-t 분포로 계산한다. 이 분포는 표준정규분포보다 양쪽 꼬리가 두터워서, 축소 공간에서 멀리 떨어진 점들을 더 잘 벌려 보이게 한다.

```text
p_{j|i} =
exp(-||x_i - x_j||^2 / (2 sigma_i^2))
/ sum_{k != i} exp(-||x_i - x_k||^2 / (2 sigma_i^2))

p_ij = (p_{j|i} + p_{i|j}) / (2n)

q_ij =
(1 + ||y_i - y_j||^2)^(-1)
/ sum_{k != l} (1 + ||y_k - y_l||^2)^(-1)
```

t-SNE의 손실함수는 원래 공간 이웃 확률 `P`와 축소 공간 이웃 확률 `Q` 사이의 Kullback-Leibler divergence다. `KL(P || Q) = sum_{i != j} p_ij log(p_ij / q_ij)`를 줄이도록 경사하강법으로 낮은 차원 좌표를 찾는다. 그래서 MNIST 손글씨 숫자처럼 복잡한 고차원 데이터를 2차원으로 볼 때 클러스터 구조가 잘 드러나지만, 계산비용이 높고 하이퍼파라미터에 민감하며 초기값에 따라 결과가 달라져 재현성이 낮을 수 있다. 또한 학습된 그림은 주어진 데이터의 시각화에 가깝고, 새 값을 안정적으로 변환하는 일반 예측 모델처럼 쓰기는 어렵다.

t-SNE 그림에서는 축의 숫자, 군집 사이의 절대 거리, 군집 크기를 과하게 해석하지 않는다. 가까운 이웃이 비슷하게 모였는지를 보는 도구에 가깝다. 같은 데이터도 perplexity나 초기값을 바꾸면 그림이 달라질 수 있으므로, 시각화 결과는 탐색적 근거로 사용한다.

```text
KL(P || Q) = sum_{i != j} p_ij log(p_ij / q_ij)
dL / dy_i = 4 * sum_j (p_ij - q_ij)(y_i - y_j)(1 + ||y_i - y_j||^2)^(-1)
```

PLS는 PCA와 달리 y를 무시하지 않는다. X의 분산만 큰 방향이 아니라, X와 y가 함께 잘 움직이는 방향을 찾는다. 수치형 입력변수 `X`에서 `Y` 예측력을 최대한 유지하는 선형변환을 만들며, `X = T P^T + E`, `Y = U Q^T + F`로 쓴다. `T`와 `U`는 score 행렬, `P`와 `Q`는 loading 행렬, `E`와 `F`는 잔차행렬이다. 핵심은 `T`와 `U`의 공분산을 크게 만드는 성분을 찾는 것이다.

PLS2는 반응변수가 여러 개인 다변량 회귀 문제다. 먼저 `X`, `Y`를 중심평균화하고, `Y`의 첫 번째 주성분으로 `u`를 초기화한다. 그다음 `X_c w`와 `u`의 공분산을 키우는 `w`를 찾고 `t = X_c w`를 만든다. 이어 `c = Y_c^T t / (t^T t)`, `u = Y_c c`를 반복해 수렴시킨다. 이후 `p = X_c^T t / (t^T t)`, `q = Y_c^T t / (t^T t)`를 구하고, `X`와 `Y`에서 `t p^T`, `t q^T`로 설명된 양을 제거하는 deflation을 한다. 지정한 성분 수만큼 반복한다. PLS1은 반응변수가 하나인 경우로, `cov(Xw, y)`를 크게 하는 단위벡터 `w`를 찾는 문제로 단순화된다. 예측 목적이 강하고 입력 변수들의 다중공선성이 클 때 효과적이다.

```text
X = T P^T + E
Y = U Q^T + F

w = X_c^T u / ||X_c^T u||
t = X_c w
c = Y_c^T t / (t^T t)
u = Y_c c

p = X_c^T t / (t^T t)
q = Y_c^T t / (t^T t)

X <- X - t p^T
Y <- Y - t q^T

B_hat = W(P^T W)^(-1)Q^T
Y_c_hat = X_c B_hat
```

## 판단 기준

1. <a id="ref-17-차원-축소"></a>[차원 축소](#note-17-차원-축소) 목적이 시각화인지, 압축인지, 예측 전처리인지 구분한다.
2. PCA 전에는 변수 스케일이 주성분을 지배하지 않도록 <a id="ref-17-표준화"></a>[표준화](#note-17-표준화)를 확인한다.
3. 설명분산비로 몇 개 성분을 남길지 판단한다.
4. t-SNE 결과의 축 숫자와 군집 간 거리를 과도하게 해석하지 않는다.
5. 지도 정보를 쓰는 LDA/PLS와 비지도 방법인 PCA를 구분한다.

## 오해와 반례

### 오해 1. PCA는 중요한 원래 변수만 고르는 방법이다.

PCA는 원래 변수의 조합으로 새 축을 만든다. 변수 선택과 다르다.

### 오해 2. t-SNE 축의 숫자는 직접 해석하면 된다.

t-SNE의 축 자체보다 가까운 점들의 국소 구조를 해석해야 한다.

### 오해 3. 차원 축소는 항상 정보 손실이 없다.

차원을 줄이면 일부 정보는 버린다. 얼마나 보존되는지 확인해야 한다.

## 예시 풀이

### 예시 1. 설문 문항 50개 줄이기

서로 비슷한 문항들이 많다면 PCA로 몇 개의 성분으로 요약해 전체 경향을 볼 수 있다.

### 예시 2. 손글씨 데이터를 2차원으로 보기

t-SNE는 고차원 이미지 벡터의 이웃 구조를 2차원에 보여 주어 숫자별 군집을 시각화할 수 있다.

## 오늘의 요약 5줄

1. 차원 축소는 많은 변수의 정보를 가능한 보존하면서 더 적은 축으로 표현하는 방법이다.
2. PCA의 주성분은 원래 변수 하나가 아니라 여러 변수를 섞은 새 방향이다.
3. SVD는 행렬을 중요한 구조로 분해해 압축과 추천 등에 쓰인다.
4. t-SNE는 가까운 이웃 구조를 시각화하는 데 강하지만 축 자체의 의미는 약하다.
5. 차원을 줄이면 해석과 계산은 쉬워지지만 정보 손실 가능성은 항상 남는다.

## 확인 문제

1. PCA의 주성분이 무엇인지 설명하라.
2. PCA에서 설명분산비를 보는 이유를 설명하라.
3. PCA 전에 표준화가 필요한 이유를 설명하라.
4. SVD가 차원 축소와 연결되는 이유를 설명하라.
5. t-SNE 결과를 해석할 때 조심할 점을 설명하라.
6. LDA와 PCA의 차이를 설명하라.
7. 왜 PCA는 원래 변수 하나를 고르는 방법이 아닌가?
8. 왜 t-SNE 그림에서 군집 사이 거리를 과하게 해석하면 위험한가?
9. Metric MDS와 Non-metric MDS의 차이를 설명하라.
10. PLS가 PCA와 달리 지도 정보를 사용하는 방식이 무엇인지 설명하라.
11. PCA에서 공분산행렬의 고윳값과 고유벡터가 각각 무엇을 의미하는지 설명하라.
12. SVD로 PCA를 계산할 때 `X_c = U Sigma V^T`와 고윳값의 관계를 설명하라.
13. 고전적 MDS에서 `-1/2 H D^2 H`가 하는 역할을 설명하라.
14. t-SNE가 축소 공간에서 표준정규분포가 아니라 자유도 1의 Student-t 분포를 쓰는 이유를 설명하라.
15. LDA의 Fisher criterion이 무엇을 크게 또는 작게 만들려는지 설명하라.
16. PLS2의 deflation 단계가 무엇을 제거하는지 설명하라.
17. 상황별 차원 축소법에서 “거리관계를 유지하여 예측력 강화”에 t-SNE가 부적합한 이유를 설명하라.
18. t-SNE의 `p_ij`, `q_ij`, KL 손실과 gradient 식을 설명하라.
19. PLS2의 `w`, `t`, `c`, `p`, `q`, deflation, 회귀계수 식을 순서대로 설명하라.
20. PCA 전에 표준화를 고려해야 하는 이유를 분산과 단위 관점에서 설명하라.
21. PCA 주성분을 원래 변수 하나처럼 해석하면 왜 위험한지 설명하라.
22. t-SNE 결과에서 축 숫자와 군집 간 거리를 과해석하면 안 되는 이유를 설명하라.
