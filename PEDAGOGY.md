# 학습 설계 원칙

이 저장소의 대상 학습자는 **데이터 사이언스, 통계, 머신러닝을 처음 접하는 사람**입니다. PDF의 차시 순서는 유지하되, 설명은 기초가 없는 상태를 기준으로 다시 씁니다.

## 작성 기준

어려운 말을 없애는 것이 목표가 아닙니다. 어려운 말을 **처음 만나는 순서**를 바꾸는 것이 목표입니다.

나쁜 순서:

```text
정의 → 약어 → 수식 → 예외
```

좋은 순서:

```text
상황 → 쉬운 말 → 그림/예시 → 정확한 용어 → 판단 기준 → 확인 문제
```

## 누적 피드백과 적용 방향

이 저장소의 작성 방향은 다음 피드백을 기준으로 고정합니다.

| 피드백 | 적용 방향 |
|---|---|
| 실습보다 이론 지식이 필요하다. | 실습을 본문 중심에 두지 않고, 개념·이유·판단 기준을 먼저 설명한다. |
| DS 커리큘럼은 기초가 없는 학생이 보는 자료다. | 난이도를 무작정 올리지 않고, 낯선 개념을 본문 주석으로 쉬운 말부터 연결한다. |
| 결론보다 “왜 그런지” 자세히 알 때 이해가 된다. | 핵심 결론마다 관찰되는 현상, 작동하는 이유, 결과가 움직이는 방향을 설명한다. |
| 영어 용어 자체의 뉘앙스가 장벽이 된다. | 영어 용어는 번역만 하지 않고, 왜 그런 이름이 붙었는지 먼저 설명한다. |
| 이미지는 그래프나 도식 위주가 좋다. | 표는 Markdown으로 만들고, 그래프·분포·흐름도 이미지만 관련 설명 근처에 배치한다. |
| 강의보다 내용이 많이 없는 느낌이다. | 원본 PDF의 개념·비교·문제 유형을 누락하지 않도록 [PDF 원본 포함 범위](SOURCE_COVERAGE.md)를 기준으로 각 차시를 점검한다. |
| 강의에 있는 내용은 다 포함되어야 한다. | 원본 문장을 베끼지 않고, 모든 원본 주제를 쉬운 설명·심화 박스·예시 풀이·확인 문제 중 한 곳 이상에 반영한다. |

## 가장 중요한 설명 원칙: 이유를 말한다

학습자가 가장 잘 이해하는 순간은 “정답”을 들었을 때가 아니라, **왜 그 정답이 그렇게 될 수밖에 없는지**를 들었을 때입니다.

예를 들어 평균, 중앙값, 최빈값을 설명할 때 다음처럼 쓰지 않습니다.

```text
오른쪽 꼬리가 긴 분포에서는 평균 > 중앙값 > 최빈값이다.
```

대신 다음 흐름으로 설명합니다.

```text
1. 히스토그램에서 대부분의 값은 왼쪽 봉우리 근처에 몰려 있다.
2. 그런데 오른쪽에 아주 큰 값들이 길게 늘어져 있다.
3. 평균은 모든 값을 더해서 나누므로, 오른쪽의 큰 값들에게 끌려간다.
4. 중앙값은 순서상 가운데 값이라 극단적으로 큰 값 몇 개에는 덜 움직인다.
5. 최빈값은 가장 많이 모인 봉우리의 위치라 꼬리 쪽으로 거의 가지 않는다.
6. 그래서 위치가 최빈값 → 중앙값 → 평균 순서가 된다.
```

모든 핵심 개념은 가능하면 다음 네 문장을 포함합니다.

- 무엇이 관찰되는가?
- 어떤 힘이나 규칙 때문에 그렇게 되는가?
- 그래서 결과가 어디로 움직이는가?
- 어떤 경우에는 이 설명이 깨지는가?

## 영어 용어는 이름의 뉘앙스를 먼저 푼다

처음 배우는 사람에게는 개념보다 **영어 이름 자체**가 먼저 장벽이 될 수 있습니다. 따라서 새 영어 용어는 번역만 붙이지 않고, 왜 그런 이름이 붙었는지 먼저 설명합니다.

예:

| 용어 | 이름을 먼저 풀어 설명하기 |
|---|---|
| Encoding | 값을 컴퓨터가 읽을 수 있는 `code`로 바꾸는 일 |
| Binning | 연속적인 값을 여러 `bin`, 즉 상자에 나누어 담는 일 |
| Smoothing | 들쭉날쭉한 값을 전체 평균 쪽으로 부드럽게 만드는 일 |
| Isolation Forest | 점을 혼자 떨어뜨려 `isolate`하는 나무를 많이 만든 숲 |
| Random Variable | 결과가 우연에 따라 달라지는 실험의 결과를 숫자로 적은 변수 |

용어 설명은 다음 순서를 따릅니다.

```text
영어 단어의 느낌 → 쉬운 한국어 → 왜 그런 이름인가 → 정확한 정의 → 주의점
```

## 차시 구성

각 차시는 다음 흐름을 기본으로 합니다.

### 1. 오늘의 한 줄

그 차시를 한 문장으로 요약합니다. 학습자가 길을 잃었을 때 돌아올 기준점입니다.

### 2. 오늘 반드시 이해할 3가지

너무 많은 목표를 주지 않습니다. 처음 보는 학습자는 세 가지 이상을 한 번에 붙잡기 어렵습니다.

### 3. 본문 안 개념 주석

영어 약어, 통계 용어, 모델 이름을 별도 단어장에 몰아두지 않습니다. 본문에서 처음 의미 있게 등장하는 단어에 주석 링크를 달고, 문서 하단의 `개념 주석`에서 쉬운 뜻과 이름의 뉘앙스를 확인하게 합니다.

`이 차시 전에 알면 좋은 것`, `학습 우선순위`, `먼저 잡는 직관`처럼 개요 역할을 하는 목록은 이미 짧은 설명을 함께 제공하므로 하단 주석으로 보내지 않습니다. 개요에만 등장하지만 이후 이해의 전제가 되는 단어는 괄호 안에 `처음 설명된 차시` 직접 링크를 둡니다. 본문에서 실제 설명이나 판단에 다시 쓰이는 단어만 주석 왕복 링크를 답니다.

본문 문장이 이미 용어를 설명하는 정의문이면 그 용어 자체에는 주석 링크를 달지 않습니다. 예를 들어 `표본 추출은 모집단에서 표본을 선택하는 과정이다`처럼 문장 안에서 뜻을 바로 풀고 있다면 `표본 추출`은 plain text로 둡니다. 영어 이름의 뉘앙스가 꼭 필요하면 하단 주석으로 보내기보다 그 문장 근처에서 직접 풀어 줍니다.

바로 위 표나 가까운 문단에서 이미 용어의 뜻을 설명했다면, 이어지는 설명 문단의 같은 용어에도 주석 링크를 반복하지 않습니다. 예를 들어 표에서 `왜도 = 좌우 비대칭 정도`를 설명했다면 다음 문장의 `왜도는 세제곱을 사용한다`는 plain text로 둡니다.

예:

```markdown
... 일부 [표본](#note-21-표본)으로 기울기를 추정한다.

## 개념 주석

- [표본](#ref-21-표본): 전체 대신 관찰한 일부 대상. ([처음 설명된 차시](../04-statistics-probability/README.md#2-모집단과-표본))
```

주석의 용어를 누르면 본문 표시 위치로 돌아오게 합니다. 이전 차시에서 이미 배운 개념은 `처음 설명된 차시` 링크를 함께 둡니다. 영어 용어가 나오면 가능하면 이름의 뜻도 주석 안에서 함께 풉니다.

### 4. 그림과 직관

표는 Markdown으로 설명하고, 이미지는 흐름도·그래프·도식처럼 **눈으로 이해해야 하는 것**에만 사용합니다. 이미지는 관련 개념 바로 위나 아래에 둡니다.

### 5. 핵심 설명

본문은 다음 순서를 지킵니다.

1. 이 개념이 왜 필요한가?
2. 쉬운 예로 보면 무엇인가?
3. 정확한 용어로 부르면 무엇인가?
4. 왜 그런 결과가 나오는가?
5. 잘못 쓰면 어떤 문제가 생기는가?

수식은 꼭 필요할 때만 넣고, 기호보다 의미를 먼저 설명합니다.

### 6. 예시 풀이

확인 문제 전에 쉬운 풀이 예시를 둡니다. 학습자는 “어떤 길이의 답을 써야 하는지”를 먼저 봐야 합니다.

### 7. 확인 문제와 해설

확인 문제는 설명형으로 두되, 별도 `answers.md`를 제공합니다. 처음 학습자는 문제만 있으면 막히기 쉽습니다.

### 8. 원본 포함 점검

각 차시 작성 후 [PDF 원본 포함 범위](SOURCE_COVERAGE.md)의 해당 차시 항목을 확인합니다. 본문이 초심자용으로 쉬워졌더라도 원본의 개념이 빠지면 완료로 보지 않습니다.

원본 주제는 다음 중 한 방식으로 반드시 반영합니다.

1. 핵심 흐름에 필요한 내용은 본문 설명으로 넣습니다.
2. 처음부터 유도하기 어려운 내용은 `심화`로 표시합니다.
3. 계산이나 선택 기준은 예시 풀이와 확인 문제 해설에서 다룹니다.
4. 종합문제는 별도 복습 차시에서 여러 차시를 연결해 다시 묻습니다.

## 심화 내용 표시

LOF, Isolation Forest, SVM의 쌍대문제처럼 처음부터 완전히 이해하기 어려운 내용은 `심화`로 표시합니다.

심화 내용의 목표는 “완벽한 유도”가 아니라 다음 두 가지입니다.

- 이름을 들었을 때 겁먹지 않기
- 지금 단계에서 알아야 할 직관만 가져가기
