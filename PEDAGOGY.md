# 학습 설계 원칙

현재 버전의 목표는 실습량을 늘리는 것이 아니라 **이론 해상도**를 높이는 것입니다. 학습자는 코드 실행 전에 개념의 조건과 한계를 먼저 설명할 수 있어야 합니다.

## 차시 구성

각 차시는 같은 6개 블록으로 구성합니다.

### 1. 학습 목표

그 차시를 마치면 무엇을 구분하고 설명할 수 있어야 하는지 명시합니다. “해 본다”보다 “판단한다”, “유도한다”, “구분한다”를 우선합니다.

### 2. 개념 지도

주요 개념의 관계를 먼저 잡습니다. 용어를 나열하지 않고, 상위 개념과 하위 개념, 전제와 결과를 연결합니다.

### 3. 핵심 이론

정의, 수식, 알고리즘을 다룹니다. 수식은 계산법보다 의미를 먼저 설명합니다.

- 각 기호가 무엇을 나타내는지 밝힙니다.
- 식이 어떤 가정을 포함하는지 적습니다.
- 같은 식이 어떤 상황에서 깨지는지 함께 적습니다.

### 4. 판단 기준

실무에서 방법을 고르는 기준을 정리합니다.

- 어떤 데이터 구조에 적합한가?
- 어떤 가정을 점검해야 하는가?
- 어떤 지표나 진단으로 실패를 발견하는가?
- 비슷한 방법과 비교했을 때 무엇을 포기하고 무엇을 얻는가?

### 5. 오해와 반례

고급 이론 학습의 핵심은 반례입니다. 각 차시는 최소 3개의 오해를 포함합니다. 예를 들어 “p-value가 작으면 효과가 크다”, “정규화는 항상 좋다”, “딥러닝은 전처리가 덜 필요하다” 같은 문장을 반박합니다.

### 6. 확인 문제

객관식보다 설명형 문제를 우선합니다. 계산 문제는 수식을 이해했는지 확인하는 수준으로 제한합니다.

## 실습의 위치

실습은 다음 목적일 때만 사용합니다.

- 수식이 실제 숫자로 어떻게 움직이는지 확인
- 가정이 깨졌을 때 결과가 어떻게 변하는지 비교
- 모델 진단 지표를 해석하는 연습

반대로, “라이브러리 사용법 익히기” 자체는 현재 커리큘럼의 중심이 아닙니다.

## 난이도 기준

난이도를 올린다는 것은 더 많은 용어를 넣는 것이 아닙니다. 다음 요소를 포함할 때 난이도가 올라갑니다.

- 전제를 명시한다.
- 수식을 해석한다.
- 비슷한 개념을 구분한다.
- 실패 사례와 반례를 든다.
- 모델 선택의 trade-off를 말한다.
