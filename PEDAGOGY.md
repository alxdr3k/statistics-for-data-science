# 학습 설계 원칙

이 저장소의 대상 학습자는 **데이터 사이언스, 통계, 머신러닝을 처음 접하는 사람**입니다. PDF의 차시 순서는 유지하되, 설명은 기초가 없는 상태를 기준으로 다시 씁니다.

## 작성 기준

어려운 말을 없애는 것이 목표가 아닙니다. 어려운 말을 **처음 만나는 순서**를 바꾸는 것이 목표입니다.

나쁜 순서:

```text
정의 → 약어 → 수식 → 예외
```

좋은 순서:

```text
상황 → 쉬운 말 → 그림/예시 → 정확한 용어 → 판단 기준 → 확인 문제
```

## 차시 구성

각 차시는 다음 흐름을 기본으로 합니다.

### 1. 오늘의 한 줄

그 차시를 한 문장으로 요약합니다. 학습자가 길을 잃었을 때 돌아올 기준점입니다.

### 2. 오늘 반드시 이해할 3가지

너무 많은 목표를 주지 않습니다. 처음 보는 학습자는 세 가지 이상을 한 번에 붙잡기 어렵습니다.

### 3. 처음 보는 단어

영어 약어, 통계 용어, 모델 이름을 바로 던지지 않습니다. 먼저 쉬운 한국어 설명을 붙입니다.

예:

| 용어 | 먼저 이해할 말 |
|---|---|
| 결측치 | 비어 있는 값 |
| 이상치 | 전체 흐름에서 유난히 튀는 값 |
| 대치 | 빈 값을 다른 값으로 채우는 일 |

### 4. 그림과 직관

표는 Markdown으로 설명하고, 이미지는 흐름도·그래프·도식처럼 **눈으로 이해해야 하는 것**에만 사용합니다. 이미지는 관련 개념 바로 위나 아래에 둡니다.

### 5. 핵심 설명

본문은 다음 순서를 지킵니다.

1. 이 개념이 왜 필요한가?
2. 쉬운 예로 보면 무엇인가?
3. 정확한 용어로 부르면 무엇인가?
4. 잘못 쓰면 어떤 문제가 생기는가?

수식은 꼭 필요할 때만 넣고, 기호보다 의미를 먼저 설명합니다.

### 6. 예시 풀이

확인 문제 전에 쉬운 풀이 예시를 둡니다. 학습자는 “어떤 길이의 답을 써야 하는지”를 먼저 봐야 합니다.

### 7. 확인 문제와 해설

확인 문제는 설명형으로 두되, 별도 `answers.md`를 제공합니다. 처음 학습자는 문제만 있으면 막히기 쉽습니다.

## 심화 내용 표시

LOF, Isolation Forest, SVM의 쌍대문제처럼 처음부터 완전히 이해하기 어려운 내용은 `심화`로 표시합니다.

심화 내용의 목표는 “완벽한 유도”가 아니라 다음 두 가지입니다.

- 이름을 들었을 때 겁먹지 않기
- 지금 단계에서 알아야 할 직관만 가져가기
