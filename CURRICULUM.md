# DS3 심화이론 커리큘럼

이 커리큘럼은 PDF 강의자료의 22차시 흐름을 기준으로 다시 설계했습니다. 목표는 실습 절차를 익히는 것이 아니라, 데이터 분석과 머신러닝 모델을 **언제, 어떤 전제 아래, 어떤 위험을 감수하고** 사용하는지 설명할 수 있게 만드는 것입니다.

## 전체 맵

| 차시 | 주제 | 핵심 질문 | 난이도 초점 |
|---:|---|---|---|
| 1 | 데이터의 이해 | 데이터는 어떤 구조와 측정 수준을 가지는가? | KDD, CRISP-DM, 변수, 시각화 문법 |
| 2 | 데이터 정제 | 결측과 이상치는 왜 단순 삭제하면 안 되는가? | MCAR/MAR/MNAR, 대치, 이상치 탐지 |
| 3 | 데이터 변환 | 변환은 왜 모델의 가설공간을 바꾸는가? | 스케일링, 인코딩, 분포 변환, 누수 |
| 4 | 통계와 확률 | 표본에서 모집단을 말하려면 무엇이 필요한가? | 표본추출, 기술통계, 확률 규칙 |
| 5 | 확률 분포 | 분포는 데이터를 어떻게 생성하는 모델인가? | 확률변수, 기대값, 분산, 대표 분포 |
| 6 | 가설 검정 | p-value는 정확히 무엇의 확률인가? | 귀무/대립가설, 검정통계량, 오류 |
| 7 | 점 추정과 구간 추정 | 좋은 추정량은 어떤 성질을 가져야 하는가? | 불편성, 효율성, MLE, 신뢰구간 |
| 8 | 고급 추정법 | 숨은 변수가 있거나 적분이 어려우면 어떻게 추정하는가? | EM, MCMC, 몬테카를로 사고 |
| 9 | 선형회귀 분석 | 선형 모델은 무엇을 설명하고 무엇을 가정하는가? | OLS, 제곱합, 진단, 계수 검정 |
| 10 | 연관성 분석 | 관계와 차이를 어떤 검정으로 판단하는가? | 상관, 정규성, 등분산, ANOVA, 범주형 검정 |
| 11 | 머신 러닝 | 학습이란 손실을 줄이는 것인가, 일반화를 얻는 것인가? | 지도/비지도, 손실, 지표, 과적합 |
| 12 | 모수적 모델 | 고정된 함수형 모델은 왜 강하고 왜 위험한가? | 가중치 모델, 규제, 나이브 베이즈 |
| 13 | 비모수적 모델 | 데이터가 모델 구조를 직접 만들 때 무엇이 달라지는가? | 트리, KNN, 불순도, 가지치기 |
| 14 | SVM | 최대 마진은 왜 일반화와 연결되는가? | 마진, 쌍대문제, KKT, 커널 |
| 15 | 앙상블 모델 | 여러 약한 모델을 합치면 왜 강해지는가? | 배깅, 랜덤 포레스트, 부스팅, 보팅 |
| 16 | 군집화 | 정답 없이 구조를 찾는다는 말은 무엇인가? | 거리, k-means, k-medoids, GMM |
| 17 | 차원 축소 | 정보 손실을 감수하고 차원을 줄이는 기준은 무엇인가? | PCA, SVD, MDS, 설명분산 |
| 18 | 일반화 기법 | 검증 성능을 높이는 절차가 왜 편향을 만들 수 있는가? | 탐색, 속성 선택, 불균형, 임계점 |
| 19 | 퍼셉트론 | 선형 분류기의 표현력은 어디서 막히는가? | 뉴런, 계단함수, XOR, MLP |
| 20 | 신경망의 구성 | 레이어는 어떤 데이터 구조를 가정하는가? | Dense, Embedding, CNN, RNN, LSTM |
| 21 | 경사 하강법 심화 | 최적화 알고리즘은 왜 학습 결과를 바꾸는가? | SGD, 미니배치, 모멘텀, Adam |
| 22 | 심층 신경망 | 깊은 네트워크는 어떤 구조적 아이디어로 안정화되는가? | AlexNet, VGG, Inception, ResNet, BatchNorm |

## 권장 학습 순서

1. **데이터와 통계 기반**: 1~5차시
2. **통계적 추론**: 6~10차시
3. **머신러닝 모델론**: 11~18차시
4. **신경망과 최적화**: 19~22차시

앞 차시의 개념이 뒤 차시의 전제가 됩니다. 예를 들어 6차시 가설 검정을 모르면 9차시 회귀 계수 검정과 10차시 ANOVA의 의미가 약해지고, 5차시 분포를 모르면 8차시 EM과 GMM을 이해하기 어렵습니다.

## 산출물 기준

각 차시를 마친 뒤 다음을 말로 설명할 수 있어야 합니다.

- 이 방법이 해결하는 문제
- 필요한 전제와 가정
- 핵심 수식의 각 항이 의미하는 것
- 실무에서 틀리기 쉬운 해석
- 비슷한 방법과의 선택 기준
